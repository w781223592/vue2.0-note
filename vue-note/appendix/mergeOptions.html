# 合并options

Vue初始化过程中有合并options的过程，让我们看看mergeOptions方法做了些什么

vm.$options = mergeOptions(
  resolveConstructorOptions(vm.constructor),
  options || {},
  vm
)

mergeOptions第一个参数是一个resolveConstructorOptions方法，定义在当前文件下

export function resolveConstructorOptions(Ctor: Class<Component>) {
  let options = Ctor.options
  if (Ctor.super) {
    const superOptions = resolveConstructorOptions(Ctor.super)
    const cachedSuperOptions = Ctor.superOptions
    if (superOptions !== cachedSuperOptions) {
      // super option changed,
      // need to resolve new options.
      Ctor.superOptions = superOptions
      // check if there are any late-modified/attached options (#4976)
      const modifiedOptions = resolveModifiedOptions(Ctor)
      // update base extend options
      if (modifiedOptions) {
        extend(Ctor.extendOptions, modifiedOptions)
      }
      options = Ctor.options = mergeOptions(superOptions, Ctor.extendOptions)
      if (options.name) {
        options.components[options.name] = Ctor
      }
    }
  }
  return options
}

它接收的参数是一个构造器，以Vue的构造函数为例，当然大部分情况下这个参数都是Vue的构造函数，如果使用Vue.extend()创造一个子类并new 一个子类时，那么这个参数就是Vue子类了，这部分内容将会在后面分析extend时详细分析。

如果我们已经深入学习到了3.3小节合并选项章节的话，我们可以继续往下深入了解一下，否则我就简单的认为resolveConstructorOptions功能就是返回一个options， 这里就是返回当前的options，接下来的内容将会默认你已经学习到第三章了。

当Ctor是Vue构造器时其实就是简单的返回Vue.options，因为子类构造器才会有super属性，当它是子类构造器时它又递归的调用了resolveConstructorOpitons，这时的参数是父级构造器，之后代码还有一些关于父类的options属性是否被改变过的判断和操作，特别需要主要下面这段话：

// check if there are any late-modified/attached options (#4976)
const modifiedOptions = resolveModifiedOptions(Ctor)

需要注意的是注释这段话，可以根据括号内的issue索引去搜一下相关的问题，这句代码是用来解决使用vue-hot-reload-api或者vue-loader时产生的一个bug，大概意思是我们利用这些插件在创建子构造器后了改变了选项，这些改变都被丢弃了。所以这段逻辑是找出改变的选项配置。

我们来详细分析一下resolveModifiedOptions方法，首先定义变量modified但不赋值，定义latest引用当前的构造器的options。不知道我们是否记得我们在3.1小节讲解Vue.extend时候，我们定义了sealedOptions，顾名思义密封的选项，我们再扒出extend的代码

Sub.options = mergeOptions(
      Super.options,
      extendOptions
)
Sub.superOptions = Super.options
Sub.sealedOptions = extend({}, Sub.options)

可见它保存的是extend({}, Sub.options), Sub.options则是mergeOptions合并的选项，合并参数是父级options和自身传入的options，extend方法就是简单做了两个对象的合并，这里不知道为什么不使用Object.assign方法，因为看起来他们是一样的，但是用in 方法遍历对象是会读原型上的属性，而对象assign方法则不会。

export function extend (to: Object, _from: ?Object): Object {
  for (const key in _from) {
    to[key] = _from[key]
  }
  return to
}

我们继续往下分析，sealedOptions是建立子构造器时候的options选项，而最新的选项latest则是Ctor.options，最后通过for in循环对比新旧选项的差异属性，然后用modified来保存被修改的属性，最后返回modified，也就是由被修改属性组成的对象。
