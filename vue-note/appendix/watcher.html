# watcher

Watcher的类定义在observer/watcher.js文件下

export default class Watcher {
  vm: Component;
  expression: string;
  cb: Function;
  id: number;
  deep: boolean;
  user: boolean;
  lazy: boolean;
  sync: boolean;
  dirty: boolean;
  active: boolean;
  deps: Array<Dep>;
  newDeps: Array<Dep>;
  depIds: SimpleSet;
  newDepIds: SimpleSet;
  before: ?Function;
  getter: Function;
  value: any;

  constructor (
    vm: Component,
    expOrFn: string | Function,
    cb: Function,
    options?: ?Object,
    isRenderWatcher?: boolean // 是否是渲染water
  ) {
    this.vm = vm
    if (isRenderWatcher) {
      vm._watcher = this
    }
    vm._watchers.push(this)
    // options
    if (options) {
      this.deep = !!options.deep // Boolean
      this.user = !!options.user
      this.lazy = !!options.lazy
      this.sync = !!options.sync
      this.before = options.before
    } else {
      this.deep = this.user = this.lazy = this.sync = false
    }

可以看到Watcher是一个比较大的一个类，在创建其实例的时候可以传入五个参数，分别是组件对象实例vm、要观察的表达式expOrFn、当被观察表达式的值变化时的回调函数cb、传递给当前观察者对象的选项options，是否是渲染watcher的标识isRenderWatcher，其constructor方法首先将vm赋值给this.vm， 如果是一个渲染watcher则使vm._watcher等于this，执行vm._watcher.push(this)，也就是说所有属于该组件实例的观察者都会被添加到该组件实例对象的vm._watcher数组中（包括渲染函数和非渲染函数的观察者）。另外该数组是在initState函数中初始化的。 再往下是对options的if...else判断，如果没有传options，则将this.deep、this.user、this.lazy、this.sync都置为false，如果传递了options参数，则通过options的同名属性的真假值对这几个参数进行初始化，下面我们来看看这几个参数的意义：

  1. options.deep: 用来告诉观察者是否是深度观测 我们平时使用Vue的watch选项或者vm.$watch函数去观测某个数据时，可以通过设置deep属性为true来深度观测数据。
  2. options.user：用来标识当前观测者的实例对象是开发者定义的还是内部定义的 等我们分析到watch选项和vm.$watch具体实现的时候，会知道除了内部定义的观察者（如：渲染函数的观察者和计算属性的观察者）之外，所有的观察者都是开发者定义的，这时该值会自动的设置为true。
  3. options.lazy：是否为惰性求值，会在计算属性中用到。
  4. options.sync：用来告诉观察者当数据变化时是否同步求值并执行回调。 默认不同步执行，而是将需要重新求值和执行回调的观察者放到一个异步队列中，当所有变化结束后统一求值并执行回调，这么做的好处我们之后分析。
  5. options.before：可以理解为Watcher实例的钩子，当数据变化触发更新前，调用在创建渲染函数的观察者实例对象时传递的before选项。
    this.cb = cb
    this.id = ++uid // uid for batching
    this.active = true
    this.dirty = this.lazy // for lazy watchers
    this.deps = []
    this.newDeps = []
    this.depIds = new Set()
    this.newDepIds = new Set()
    this.expression = process.env.NODE_ENV !== 'production'
      ? expOrFn.toString()
      : ''

再往下就是上面这段代码this.active标识着观察者实例对象是否是激活状态，然后让this.dirty = this.lazy，标识是否是一个惰性观察者，之后的四个初始化属性deps、newDeps、depIds、newDepIds是用来实现避免收集重复依赖，之后在非生产环境设置this.expression为expOrFn函数的字符串表示，所以它是在开发环境使用的。

    // parse expression for getter
    if (typeof expOrFn === 'function') {
      this.getter = expOrFn
    } else {
      this.getter = parsePath(expOrFn)
      if (!this.getter) {
        this.getter = noop
        process.env.NODE_ENV !== 'production' && warn(
          `Failed watching path: "${expOrFn}" ` +
          'Watcher only accepts simple dot-delimited paths. ' +
          'For full control, use a function instead.',
          vm
        )
      }
    }
    this.value = this.lazy ? undefined
      : this.get()
  }

接下来就判断了expOrFn是否是函数类型，是函数类型则把它赋值给this.getter,否则使this.getter 等于 parsePath(expOrFn)，赋值后的getter为空时将getter设置为空函数并在非生产环境报警告。 先来看parsePath(expOrFn)，该函数定义在core/util/lang.js下

export const unicodeRegExp = /a-zA-Z\u00B7\u00C0-\u00D6\u00D8-\u00F6\u00F8-\u037D\u037F-\u1FFF\u200C-\u200D\u203F-\u2040\u2070-\u218F\u2C00-\u2FEF\u3001-\uD7FF\uF900-\uFDCF\uFDF0-\uFFFD/
const bailRE = new RegExp(`[^${unicodeRegExp.source}.$_\\d]`)
export function parsePath (path: string): any {
  if (bailRE.test(path)) {
    return
  }
  const segments = path.split('.')
  return function (obj) {
    for (let i = 0; i < segments.length; i++) {
      if (!obj) return
      obj = obj[segments[i]]
    }
    return obj
  }
}

可以看到其定义了一个复杂的正则表达式，目的是为了匹配合法的path，path不能含有上述正则定义之外的字符串（比如/、*之类），如果含有不合法字符串，则函数第一步检查为true，则直接return，所以正则定义的是合法字符串，但是是通过反向检查去过滤的，因为bailRE定义是^开头的所以是检查是否包含这些字符以外的特殊字符。之后定义segments变量为path通过.分割的数组，最后返回一个function实现一个闭包函数，内层函数接收obj参数，首先遍历segments，如果!obj直接return，否则让obj等于obj中的segments[i]，这可以做到对对象中的某个属性进行监听，比如obj.a.b，参考我们使用$watch函数的例子：

分析到这里已经豁然开朗了，这个方法就是把一个对象访问的一个字符串表达式给转化成函数了并且返回该对象属性的访问，同时又触发数据属性get拦截。接下来就是对this.value赋值，如果this.lazy存在就返回undefined不存在则执行get()方法，接下来我们分析get()方法。

  /**
   * Evaluate the getter, and re-collect dependencies.
   */
  get () {
    pushTarget(this)
    let value
    const vm = this.vm
    try {
      value = this.getter.call(vm, vm)
    } catch (e) {
      if (this.user) {
        handleError(e, vm, `getter for watcher "${this.expression}"`)
      } else {
        throw e
      }
    } finally {
      // "touch" every property so they are all tracked as
      // dependencies for deep watching
      if (this.deep) {
        traverse(value)
      }
      popTarget()
      this.cleanupDeps()
    }
    return value
  }

get()涉及比较多的响应式内容，这儿我们简单了解一下之后到了响应式章节详细分析，get方法实现的功能很简单就是：求值。最后返回value，value是通过this.getter.call(vm, vm)返回的，实际上就是执行到了之前传入的updateComponent方法。

  /**
   * Add a dependency to this directive.
   */
  addDep (dep: Dep) {
    const id = dep.id
    if (!this.newDepIds.has(id)) {
      this.newDepIds.add(id)
      this.newDeps.push(dep)
      if (!this.depIds.has(id)) {
        dep.addSub(this)
      }
    }
  }

  /**
   * Clean up for dependency collection.
   */
  cleanupDeps () {
    let i = this.deps.length
    while (i--) {
      const dep = this.deps[i]
      if (!this.newDepIds.has(dep.id)) {
        dep.removeSub(this)
      }
    }
    let tmp = this.depIds
    this.depIds = this.newDepIds
    this.newDepIds = tmp
    this.newDepIds.clear()
    tmp = this.deps
    this.deps = this.newDeps
    this.newDeps = tmp
    this.newDeps.length = 0
  }

  /**
   * Subscriber interface.
   * Will be called when a dependency changes.
   */
  update () {
    /* istanbul ignore else */
    if (this.lazy) {
      this.dirty = true
    } else if (this.sync) {
      this.run()
    } else {
      queueWatcher(this)
    }
  }

  /**
   * Scheduler job interface.
   * Will be called by the scheduler.
   */
  run () {
    if (this.active) {
      const value = this.get()
      if (
        value !== this.value ||
        // Deep watchers and watchers on Object/Arrays should fire even
        // when the value is the same, because the value may
        // have mutated.
        isObject(value) ||
        this.deep
      ) {
        // set new value
        const oldValue = this.value
        this.value = value
        if (this.user) {
          const info = `callback for watcher "${this.expression}"`
          invokeWithErrorHandling(this.cb, this.vm, [value, oldValue], this.vm, info)
        } else {
          this.cb.call(this.vm, value, oldValue)
        }
      }
    }
  }

  /**
   * Evaluate the value of the watcher.
   * This only gets called for lazy watchers.
   */
  evaluate () {
    this.value = this.get()
    this.dirty = false
  }

  /**
   * Depend on all deps collected by this watcher.
   */
  depend () {
    let i = this.deps.length
    while (i--) {
      this.deps[i].depend()
    }
  }

  /**
   * Remove self from all dependencies' subscriber list.
   */
  teardown () {
    if (this.active) {
      // remove self from vm's watcher list
      // this is a somewhat expensive operation so we skip it
      // if the vm is being destroyed.
      if (!this.vm._isBeingDestroyed) {
        remove(this.vm._watchers, this)
      }
      let i = this.deps.length
      while (i--) {
        this.deps[i].removeSub(this)
      }
      this.active = false
    }
  }
}
