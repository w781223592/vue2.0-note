# vue-router

# 第一部分 处理完整路径

getFullPath方法定义于util/route.js中。

function getFullPath (
  { path, query = {}, hash = '' },
  _stringifyQuery
): string {
  const stringify = _stringifyQuery || stringifyQuery
  return (path || '/') + stringify(query) + hash
}

如果传入自定义的stringifyQuery方法就采用自定义方法，否则采用默认的方法加密 在util/query.js中

export function stringifyQuery (obj: Dictionary<string>): string {
  const res = obj
    ? Object.keys(obj)
      .map(key => {
        const val = obj[key]

        if (val === undefined) {
          return ''
        }
        if (val === null) {
          return encode(key)
        }
        if (Array.isArray(val)) {
          const result = []
          val.forEach(val2 => {
            if (val2 === undefined) {
              return
            }
            if (val2 === null) {
              result.push(encode(key))
            } else {
              result.push(encode(key) + '=' + encode(val2))
            }
          })
          return result.join('&')
        }
        return encode(key) + '=' + encode(val)
      })
      .filter(x => x.length > 0)
      .join('&')
    : null
  return res ? `?${res}` : ''
}

看起来非常复杂，其实就是对一些参数做了处理，比如对象、数组之类，通过encode方法进行编码，其定义如下

const encodeReserveRE = /[!'()*]/g
// !'()*转化为 %21%27%28%29%2a
const encodeReserveReplacer = c => '%' + c.charCodeAt(0).toString(16)
const commaRE = /%2C/g // ,

// fixed encodeURIComponent which is more conformant to RFC3986:
// - escapes [!'()*]
// - preserve commas
const encode = str =>
  encodeURIComponent(str)
    .replace(encodeReserveRE, encodeReserveReplacer)
    .replace(commaRE, ',')

其实就是把一些特殊符号编码，这是根据RFC3986规范来的

stringifyQuery({
    foo: null,
    bar: '',
    arr: [1, null, 3]
})
 // '?foo&bar=&arr=1&arr&arr=3'

这是stringifyQuery执行与返回结果，具体实现我们就不去一一分析了。

formatMatch方法也定义于util/route.js中

function formatMatch (record: ?RouteRecord): Array<RouteRecord> {
  const res = []
  while (record) {
    res.unshift(record)
    record = record.parent
  }
  return res
}

它的定义就简单许多了，先初始化res数组，如果传入了record对象并且不为空就，record我们在8.3小节已经了解过了。这里就是把路由record以及它的父级存储到res中，直到没有父级为止，顺序为父在前，子在后。

# 第二部分 解析路径

normalizeLocation定义于util/location.js

export function normalizeLocation (
  raw: RawLocation,
  current: ?Route,
  append: ?boolean,
  router: ?VueRouter
): Location {
  let next: Location = typeof raw === 'string' ? { path: raw } : raw
  // named target
  if (next._normalized) {
    return next
  } else if (next.name) {
    next = extend({}, raw)
    const params = next.params
    if (params && typeof params === 'object') {
      next.params = extend({}, params)
    }
    return next
  }

  // relative params
  if (!next.path && next.params && current) {
    next = extend({}, next)
    next._normalized = true
    const params: any = extend(extend({}, current.params), next.params)
    if (current.name) {
      next.name = current.name
      next.params = params
    } else if (current.matched.length) {
      const rawPath = current.matched[current.matched.length - 1].path
      next.path = fillParams(rawPath, params, `path ${current.path}`)
    } else if (process.env.NODE_ENV !== 'production') {
      warn(false, `relative params navigation requires a current route.`)
    }
    return next
  }

  const parsedPath = parsePath(next.path || '')
  const basePath = (current && current.path) || '/'
  const path = parsedPath.path
    ? resolvePath(parsedPath.path, basePath, append || next.append)
    : basePath

  const query = resolveQuery(
    parsedPath.query,
    next.query,
    router && router.options.parseQuery
  )

  let hash = next.hash || parsedPath.hash
  if (hash && hash.charAt(0) !== '#') {
    hash = `#${hash}`
  }

  return {
    _normalized: true,
    path,
    query,
    hash
  }
}

首先定义了next，如果raw是string类型，就把它写成一个{ path: raw }，否则就等于raw。接下来判断如果next._normalized已经标准化过的话就直接返回next。再如果next.name存在的话就通过extend方法去浅克隆一个raw对象。extend方法我们在分析vue源码时候已经见过了，这里extend方法和vue中定义是一致的。然后还要获取next.params，显然这是在raw为一个对象时才会有效。如果params是一个object，那就再次浅克隆并赋值给next的params。接下来也是对一些参数的处理，其实这部分内容我们目前还不会走到，所以暂时还没必要去分析。
我们来直接往下看parsedPath定义，在util/path.js中

export function parsePath (path: string): {
  path: string;
  query: string;
  hash: string;
} {
  let hash = ''
  let query = ''

  const hashIndex = path.indexOf('#')
  if (hashIndex >= 0) {
    hash = path.slice(hashIndex)
    path = path.slice(0, hashIndex)
  }

  const queryIndex = path.indexOf('?')
  if (queryIndex >= 0) {
    query = path.slice(queryIndex + 1)
    path = path.slice(0, queryIndex)
  }

  return {
    path,
    query,
    hash
  }
}

首先定义hash和query然后从patch中获取#位置索引，如果index不等于-1，就将hash值设置为#和#后面字符串。path设置为#前的字符串。然后同样查找 ? 位置，并以同样方式处理query和修改后的path，但是query不包括 ? 。path则是重新赋值。最后返回path、query和hash字符串。所以parsePath主要是用来解析path的。
接着normalizeLocation方法定义基础路径basePath，然后定义最终路径path。path在parsedPath.path存在时会调用resolvePath方法处理，否则返回basePath。

export function resolvePath (
  relative: string,
  base: string,
  append?: boolean
): string {
  const firstChar = relative.charAt(0)
  if (firstChar === '/') {
    return relative
  }

  if (firstChar === '?' || firstChar === '#') {
    return base + relative
  }

  const stack = base.split('/')

  // remove trailing segment if:
  // - not appending
  // - appending to trailing slash (last segment is empty)
  if (!append || !stack[stack.length - 1]) {
    stack.pop()
  }

  // resolve relative path
  const segments = relative.replace(/^\//, '').split('/')
  for (let i = 0; i < segments.length; i++) {
    const segment = segments[i]
    if (segment === '..') {
      stack.pop()
    } else if (segment !== '.') {
      stack.push(segment)
    }
  }

  // ensure leading slash
  if (stack[0] !== '') {
    stack.unshift('')
  }

  return stack.join('/')
}

resolvePath也定义在util/path.js中。第一个参数是相对路径，第二个参数是基础路径，第三个参数这里传入的是false。
然后取相对路径第一个字符，如果是/就直接返回相对路径relative。如果是'?'或者#号则返回base + relative。接下来是出现两种情况要删除尾段：

  1. append为false
  2. stack最后一个为空，也就是尾部是/导致stack最后一个为空。

此时的操作就是stack.pop()。
接下来定义segments为relative去掉开头/并且通过/切割成数组，然后就是遍历数组，如果遍历的值为两个. 则说明是回退路径，则调用stack.pop()在base路径基础上回退一级。再如果不等于一个.，即不是当前路径，就push segment，也就是到下一级。如果stack第一个不是空字符串，那就在前面插入一个空字符串。这样最后将数组转换为字符串并用/连接时就能得到一个/开头的路径。
然后接着看normalizeLocation方法定义了query

  const query = resolveQuery(
    parsedPath.query,
    next.query,
    router && router.options.parseQuery
  )

调用resolveQuery方法，第一个传入的是parsedPath返回的query，第二个是next的query，第三个是options配置的parseQuery方法。 然后接下来看resolveQuery定义，在util/query.js中

export function resolveQuery (
  query: ?string,
  extraQuery: Dictionary<string> = {},
  _parseQuery: ?Function
): Dictionary<string> {
  const parse = _parseQuery || parseQuery
  let parsedQuery
  try {
    parsedQuery = parse(query || '')
  } catch (e) {
    process.env.NODE_ENV !== 'production' && warn(false, e.message)
    parsedQuery = {}
  }
  for (const key in extraQuery) {
    const value = extraQuery[key]
    parsedQuery[key] = Array.isArray(value)
      ? value.map(castQueryParamValue)
      : castQueryParamValue(value)
  }
  return parsedQuery
}

首先是定义parse为传入的parseQuery或默认的parseQuery。然后通过parse处理query然后保存在parsedQuery中

function parseQuery (query: string): Dictionary<string> {
  const res = {}

  query = query.trim().replace(/^(\?|#|&)/, '')
  if (!query) {
    return res
  }
  query.split('&').forEach(param => {
    const parts = param.replace(/\+/g, ' ').split('=')
    const key = decode(parts.shift())
    const val = parts.length > 0 ? decode(parts.join('=')) : null
    if (res[key] === undefined) {
      res[key] = val
    } else if (Array.isArray(res[key])) {
      res[key].push(val)
    } else {
      res[key] = [res[key], val]
    }
  })
  return res
}

parseQuery方法比较简单，首先将query首尾空格去掉，然后将开头的?、#、&字符替换为空字符串。如果这时query为空就直接返回空对象res。接下来将query通过&切割为数组并forEach遍历，把遍历项+号转化为空格，然后再通过=切割为数组保存在parts中。
如果param是id=1字符串，parts就是[id, 1]这样的数组，然后我们将parts.shift就是得到数组第一项，并且防止它是编码过的内容，还需要通过decode去解码。这里decode定义比较简单就不去专门分析了，下面是decode代码

export function decode (str: string) {
  try {
    return decodeURIComponent(str)
  } catch (err) {
    if (process.env.NODE_ENV !== 'production') {
      warn(false, `Error decoding "${str}". Leaving it intact.`)
    }
  }
  return str
}

然后就是获取val，如果parts数组不为0的话说明是有参数值得，然后再把[1]转化成字符串1进行解码，至于这里为什么join传入=号呢，那是因为参数值如果本身就是1=1这种带=号形式的话，直接join( )就得不到原来结果了。如果数组长0就赋值为null。
接下来是对res[key]是否完全等于undefined的判断，如果是则给它赋值为val，否则再判断它是不是数组，如果是数组就把val添加进去。可能看到这里我们都不太清楚这段代码有什么意义，但是看到最后else逻辑我想应该会豁然开朗吧。它就是将我们的数组query再变回数组形式，然后返回。
然后再回来看resolveQuery，最后是一个for循环，它主要是靠castQueryParamValue方法实现功能，然后我们来看castQueryParamValue

const castQueryParamValue = value => (value == null || typeof value === 'object' ? value : String(value))

其实就是extraQuery对象和parsedQuery对象做了个合并。所以resolveQuery方法主要是将stringifyQuery方法转化过的query参数进行解析恢复。
最后我们回到normalizeLocation方法中

  let hash = next.hash || parsedPath.hash
  if (hash && hash.charAt(0) !== '#') {
    hash = `#${hash}`
  }

  return {
    _normalized: true,
    path,
    query,
    hash
  }
}

最后标准化了hash的格式，然后返回path、query、hash和_normalized标识位。但是在某些情况下是直接返回next对象的，这时next会有name、params等其他属性

abort在history/base.js中的confirmTransition方法中

这个方法主要是判断是否是导航失败报错，isError定义则是

function createRouterError (from, to, type, message) {
  const error = new Error(message)
  error._isRouter = true
  error.from = from
  error.to = to
  error.type = type

  return error
}

# 第四部分 滚动行为

在transitionTo的回调方法中会对滚动进行处理，滚动处理都是通过handleScroll来处理的。

export function handleScroll (
  router: Router,
  to: Route,
  from: Route,
  isPop: boolean
) {
  if (!router.app) {
    return
  }

  const behavior = router.options.scrollBehavior
  if (!behavior) {
    return
  }
  if (process.env.NODE_ENV !== 'production') {
    assert(typeof behavior === 'function', `scrollBehavior must be a function`)
  }
  // wait until re-render finishes before scrolling
  router.app.$nextTick(() => {
    const position = getScrollPosition()
    const shouldScroll = behavior.call(
      router,
      to,
      from,
      isPop ? position : null
    )
    if (!shouldScroll) {
      return
    }
    if (typeof shouldScroll.then === 'function') {
      shouldScroll
        .then(shouldScroll => {
          scrollToPosition((shouldScroll: any), position)
        })
        .catch(err => {
          if (process.env.NODE_ENV !== 'production') {
            assert(false, err.toString())
          }
        })
    } else {
      scrollToPosition(shouldScroll, position)
    }
  })
}

先获取用户定义的滚动行为，如果用户没有定义就直接什么都不做。如果获取的behavior不是一个函数就一个错误，在页面渲染完成后去获取滚动位置

function getScrollPosition (): ?Object {
  const key = getStateKey()
  if (key) {
    return positionStore[key]
  }
}

通过当前key去获取保存滚动位置，然后执行我们定义的滚动函数，如果behavior返回的shouldScroll为空则直接返回，如过返回的是一个promise对象，则调用then方法并传入成功回调，然后这里执行一个scrollToPosition方法，我们先继续往后看，else则是如果返回不是函数同样去执行scrollToPosition。

const hashStartsWithNumberRE = /^#\d/

function scrollToPosition (shouldScroll, position) {
  const isObject = typeof shouldScroll === 'object'
  if (isObject && typeof shouldScroll.selector === 'string') {
    // getElementById would still fail if the selector contains a more complicated query like #main[data-attr]
    // but at the same time, it doesn't make much sense to select an element with an id and an extra selector
    // 如果选择器包含一个更复杂的查询，如#main[data-attr]，getElementById仍然会失败，
    // 但同时，选择一个具有id和额外选择器的元素没有多大意义
    const el = hashStartsWithNumberRE.test(shouldScroll.selector) // $flow-disable-line
      ? document.getElementById(shouldScroll.selector.slice(1)) // $flow-disable-line
      : document.querySelector(shouldScroll.selector)

    if (el) {
      let offset =
        shouldScroll.offset && typeof shouldScroll.offset === 'object'
          ? shouldScroll.offset
          : {}
      offset = normalizeOffset(offset)
      position = getElementPosition(el, offset)
    } else if (isValidPosition(shouldScroll)) {
      position = normalizePosition(shouldScroll)
    }
  } else if (isObject && isValidPosition(shouldScroll)) {
    position = normalizePosition(shouldScroll)
  }

  if (position) {
    // $flow-disable-line
    if ('scrollBehavior' in document.documentElement.style) {
      window.scrollTo({
        left: position.x,
        top: position.y,
        // $flow-disable-line
        behavior: shouldScroll.behavior
      })
    } else {
      window.scrollTo(position.x, position.y)
    }
  }
}

这个方法有点复杂，首先要确定shouldScroll是个对象，然后判断有没有selector，如果有就获取这个dom。接下来要是取到dom就去获取shouldScroll.offset，如果没有设置就初始化为空对象。
接下来就对offset做一个标准化，确保它们是能用的

function normalizeOffset (obj: Object): Object {
  return {
    x: isNumber(obj.x) ? obj.x : 0,
    y: isNumber(obj.y) ? obj.y : 0
  }
}

然后去计算一个滚动位置

function getElementPosition (el: Element, offset: Object): Object {
  const docEl: any = document.documentElement
  const docRect = docEl.getBoundingClientRect()
  const elRect = el.getBoundingClientRect()
  return {
    x: elRect.left - docRect.left - offset.x,
    y: elRect.top - docRect.top - offset.y
  }
}

因为开始left top都是0，往下滚动往右滚动会变为负值，所以都是减去它们的值。如果el不存在再去检查shouldScroll是不是有效的

function isValidPosition (obj: Object): boolean {
  return isNumber(obj.x) || isNumber(obj.y)
}

export function setupScroll () {
  // Prevent browser scroll behavior on History popstate
  // 防止浏览器在历史记录弹出状态下滚动
  if ('scrollRestoration' in window.history) {
    window.history.scrollRestoration = 'manual'
  }
  // Fix for #1585 for Firefox
  // Fix for #2195 Add optional third attribute to workaround a bug in safari https://bugs.webkit.org/show_bug.cgi?id=182678
  // Fix for #2774 Support for apps loaded from Windows file shares not mapped to network drives: replaced location.origin with
  // window.location.protocol + '//' + window.location.host
  // location.host contains the port and location.hostname doesn't
  const protocolAndPath = window.location.protocol + '//' + window.location.host
  const absolutePath = window.location.href.replace(protocolAndPath, '')
  // preserve existing history state as it could be overriden by the user
  const stateCopy = extend({}, window.history.state)
  stateCopy.key = getStateKey()
  window.history.replaceState(stateCopy, '', absolutePath)
  window.addEventListener('popstate', handlePopState)
  return () => {
    window.removeEventListener('popstate', handlePopState)
  }
}

这里监听popstate方法会调用handlePopState

function handlePopState (e) {
  saveScrollPosition()
  if (e.state && e.state.key) {
    setStateKey(e.state.key)
  }
}

这个方法主要是去重新设置滚动位置，离开本页面时先记录一次位置，然后调用setStateKey方法设置key为新页面key值，这样去获取滚动位置时候就能通过新的key值去找到新页面的滚动位置，从而保证点击浏览器前进回退时实现原生的效果。
