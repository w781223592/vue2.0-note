# 初始化proxy

initProxy方法定义于instance/proxy文件下

  initProxy = function initProxy (vm) {
    if (hasProxy) {
      // determine which proxy handler to use
      const options = vm.$options
      const handlers = options.render && options.render._withStripped
        ? getHandler
        : hasHandler
      vm._renderProxy = new Proxy(vm, handlers)
    } else {
      vm._renderProxy = vm
    }
  }

判断hasProxy是否为true，如果为true，使用new Proxy代理vm来对开发版本做一些警告处理。当使用render函数时options.render为true，_withStripped只在测试代码中被为true，handlers在正常情况为hasHandler，其定义如下：

const hasHandler = {
    has (target, key) {
      const has = key in target
      const isAllowed = allowedGlobals(key) ||
        (typeof key === 'string' && key.charAt(0) === '_' && !(key in target.$data))
      if (!has && !isAllowed) {
        if (key in target.$data) warnReservedPrefix(target, key)
        else warnNonPresent(target, key)
      }
      return has || !isAllowed
    }
  }

allowedGlobals方法定义

const allowedGlobals = makeMap(
    'Infinity,undefined,NaN,isFinite,isNaN,' +
    'parseFloat,parseInt,decodeURI,decodeURIComponent,encodeURI,encodeURIComponent,' +
    'Math,Number,Date,Array,Object,Boolean,String,RegExp,Map,Set,JSON,Intl,BigInt,' +
    'require' // for Webpack/Browserify
  )

由上可以看出，allowedGlobals方法仅仅是makeMap方法的调用，并传入一个长的字符串，看这些字符串是不是非常熟悉，没错它们都是js的全局对象，然后我们看makeMap方法的定义：

export function makeMap (
  str: string,
  expectsLowerCase?: boolean
): (key: string) => true | void {
  const map = Object.create(null)
  const list: Array<string> = str.split(',')
  for (let i = 0; i < list.length; i++) {
    map[list[i]] = true
  }
  return expectsLowerCase
    ? val => map[val.toLowerCase()]
    : val => map[val]
}

makeMap函数是一个使用闭包返回的一个新的函数，它将传入的字符串按','分割组成数组，创建空的对象map，遍历数组并将数组每一项字符串作为对象key给对象添加新属性，value都设置为true，最后返回的新函数根据传入的val判断map中是否有val属性。

这样回过头来看allowedGlobals函数可以判断给定的key是否出现在给定字符串中，这些key都是js 的原生对象，是可以在全局访问到的，这样再来看hasHandler.has方法，实质通过ES6中的Proxy对象方法拦截in操作，has方法先定义变量has = key in target，但没有直接返回has结果，而是对!has情况做了警告处理，然后我们看看下面的警告是在什么时候触发的：

if (!has && !isAllowed) {
   if (key in target.$data) warnReservedPrefix(target, key)
   else warnNonPresent(target, key)
}

上述代码是在!has和!isAllowed都成立的条件下执行的, 这就使得我们在target上也就是vm上找不到这个属性(即!has成立)也不一定会报出警告，必须要!isAllowed也成立。 而!isAllowed成立的条件需要我们先分析isAllowed是什么

const isAllowed = allowedGlobals(key) || (typeof key === 'string' && key.charAt(0) === '_' && !(key in target.$data))

之前我们已经分析过了allowedGlobals方法，它的主要防止我们在模板访问一些js原生对象时报警告，所以我们直接看后半部分逻辑，首先要存在这个key是一个字符串且它的首位一定是'_'符号，并且它没有在$data上定义过，如果在模板中访问一个这样的属性也是不会报warn的，eg：

<div id="app">{{ message }} {{ isNaN }} {{ _data }}</div>

反之我们访问一个未在data和methods下定义的属性且不是js原生对象就会触发warn，另外还对带_的属性做了特殊处理，访问_key属性是一般是不会被这部分代码拦截的，前提是这个属性不在$data上定义，首先未定义的key属性会报warnNonPresent的警告。

warnNonPresent返回的警告我们都不会陌生当我们在模板访问了一个没有在data，methods等数据项中定义的属性时就会报这个错误，这是开发过程中常遇到的。

在data中定义_key属性时若直接在模板中使用{ { _key } }会报warnReservedPrefix 这个我们开发过程中很少会碰到，它提示你必须使用{ { $data._key } }去访问。 仔细分析has函数，它只拦截了在data中定义的_key属性，这是非常不合理的，所以has函数return时有判断has || !isAllowed，这使得我们访问一个没有在任何地方定义的_key时会返回false

<div id="app">{{ _a }}</div>

所以当我这样写时会报error而不是warn，至于在哪报的错相信可以在后面的逻辑中找到答案，也许我会在分析渲染函数时找到答案。

提示

这么做是由于渲染函数中会包含很多以 _开头的内部方法，所以vue在这里特殊处理_开头的属性和方法，如之前我们将会在分析渲染函数时遇到的 _c、_v 等等

以下是两个警告函数内容：

const warnNonPresent = (target, key) => {
    warn(
      `Property or method "${key}" is not defined on the instance but ` +
      'referenced during render. Make sure that this property is reactive, ' +
      'either in the data option, or for class-based components, by ' +
      'initializing the property. ' +
      'See:  ',
      target
    )
  }

  const warnReservedPrefix = (target, key) => {
    warn(
      `Property "${key}" must be accessed with "$data.${key}" because ` +
      'properties starting with "$" or "_" are not proxied in the Vue instance to ' +
      'prevent conflicts with Vue internals. ' +
      'See: https://vuejs.org/v2/api/#data',
      target
    )
  }

另外还有proxy.js文件下还有一段逻辑

 // 浏览器是否支持Proxy语法
  const hasProxy =
    typeof Proxy !== 'undefined' && isNative(Proxy)

  if (hasProxy) {
    // isBuiltInModifier 函数用来检测是否是内置修饰符
    const isBuiltInModifier = makeMap('stop,prevent,self,ctrl,shift,alt,meta,exact')
    // 为cofig.keyCodes 设置set代理，防止内置修饰符被覆盖
    config.keyCodes = new Proxy(config.keyCodes, {
      set (target, key, value) {
        if (isBuiltInModifier(key)) {
          warn(`Avoid overwriting built-in modifier in config.keyCodes: .${key}`)
          return false
        } else {
          target[key] = value
          return true
        }
      }
    })
  }

hasProxy判断环境是否支持proxy，isBuiltInModifier函数用来检测给定的值是否是内置事件的修饰符，在Vue中我们可以通过事件修饰符很方便的做一些工作，比如阻止默认事件等等。然后给config.keyCodes设置了set代理，其目的是防止开发者在自定义键位别名的时候，覆盖了内置修饰符，eg：

Vue.config.keyCodes.shift = 16
