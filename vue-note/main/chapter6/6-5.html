# 6.5 v-model（1）

  我们都知道v-model是Vue实现双向绑定的一个能力。我们知道Vue中可以通过数据来驱动视图变化，那么视图变化能不能改变数据呢？答案是可以的，v-model就实现了这样一个过程。
  对于input这类型标签我们通过键盘输入等事件就会改变视图，如果我们通过v-model指令绑定数据，那么在视图改变时也能驱动数据的改变。如果我们调用方法去改变绑定的数据，同时视图也会发生改变，这就是所谓的双向绑定。
  对于v-model我们还是要从编译说起，parse解析到v-model指令后将其添加到指令对象el.directives中，在执行genData时首先会执行genDirectives，这就会执行到平台化的model方法，在web/compiler/directives/model.js中定义了model函数。

export default function model (
  el: ASTElement,
  dir: ASTDirective,
  _warn: Function
): ?boolean {
  warn = _warn
  const value = dir.value
  const modifiers = dir.modifiers
  const tag = el.tag
  const type = el.attrsMap.type

  if (process.env.NODE_ENV !== 'production') {
    // inputs with type="file" are read only and setting the input's
    // value will throw an error.
    if (tag === 'input' && type === 'file') {
      warn(
        `<${el.tag} v-model="${value}" type="file">:\n` +
        `File inputs are read only. Use a v-on:change listener instead.`,
        el.rawAttrsMap['v-model']
      )
    }
  }
  if (el.component) {
    genComponentModel(el, value, modifiers)
    // component v-model doesn't need extra runtime
    return false
  } else if (tag === 'select') {
    genSelect(el, value, modifiers)
  } else if (tag === 'input' && type === 'checkbox') {
    genCheckboxModel(el, value, modifiers)
  } else if (tag === 'input' && type === 'radio') {
    genRadioModel(el, value, modifiers)
  } else if (tag === 'input' || tag === 'textarea') {
    genDefaultModel(el, value, modifiers)
  } else if (!config.isReservedTag(tag)) {
    genComponentModel(el, value, modifiers)
    // component v-model doesn't need extra runtime
    return false
  } else if (process.env.NODE_ENV !== 'production') {
    warn(
      `<${el.tag} v-model="${value}">: ` +
      `v-model is not supported on this element type. ` +
      'If you are working with contenteditable, it\'s recommended to ' +
      'wrap a library dedicated for that purpose inside a custom component.',
      el.rawAttrsMap['v-model']
    )
  }
  // ensure runtime directive metadata
  return true
}

  如果是input标签并type为file时，报一个警告，让我们使用@change事件而不要用v-model，因为它应该是只读的。
接下来就是对不同标签采取不同的处理方式：

  1. 在组件上使用v-model时，会执行genComponentModel函数并且return false。对于使用is属性绑定的组件也认为在组件上使用。
  2. 如果是select标签则采用genSelect方法处理。
  3. 如果是checkbox采用genCheckboxModel方法处理。
  4. 如果是radio采用genRadioModel方法处理。
  5. 如果是input标签或者textarea标签用genDefaultModel处理。
  6. 如果是其他情况说明在不支持v-model的标签上使用，则报警告。

  只要不是在组件上使用v-model那么最后就会返回true。我们接下来先看默认情况下的处理也就是genDefaultModel方法

function genDefaultModel (
  el: ASTElement,
  value: string,
  modifiers: ?ASTModifiers
): ?boolean {
  const type = el.attrsMap.type

  // warn if v-bind:value conflicts with v-model
  // except for inputs with v-bind:type
  if (process.env.NODE_ENV !== 'production') {
    const value = el.attrsMap['v-bind:value'] || el.attrsMap[':value']
    const typeBinding = el.attrsMap['v-bind:type'] || el.attrsMap[':type']
    if (value && !typeBinding) {
      const binding = el.attrsMap['v-bind:value'] ? 'v-bind:value' : ':value'
      warn(
        `${binding}="${value}" conflicts with v-model on the same element ` +
        'because the latter already expands to a value binding internally',
        el.rawAttrsMap[binding]
      )
    }
  }
  // 省略部分
}

  首先会判断我们有没有绑定value属性，因为我们v-model实际上就是去动态绑定了value属性，如果我们同时绑定v-bind:value，那么会报一个冲突的警告，但是如果我们动态绑定了type属性则触发。

  const { lazy, number, trim } = modifiers || {}
  const needCompositionGuard = !lazy && type !== 'range'
  const event = lazy
    ? 'change'
    : type === 'range'
      ? RANGE_TOKEN
      : 'input'

  这里会获取v-model的几个专用修饰符。如果!lazy并且type不为'range'时，needCompositonGuard为true。   然后定义event，如果lazy为true，event为change，lazy为不为true则判断type为range时将event设置为'__r'，否则为input。

  let valueExpression = '$event.target.value'
  if (trim) {
    valueExpression = `$event.target.value.trim()`
  }
  if (number) {
    valueExpression = `_n(${valueExpression})`
  }

  接下来这里定义获取value的表达式，如果trim存在的话会让表达式后面加一个trim方法。如果有number修饰符，那么会使用_n方法处理

  target._n = toNumber

  其实就是通过toNumber方法去将字符串转化为数组，转化失败则返回原字符串。

  let code = genAssignmentCode(value, valueExpression)
  if (needCompositionGuard) {
    code = `if($event.target.composing)return;${code}`
  }
  addProp(el, 'value', `(${value})`)
  addHandler(el, event, code, null, true)
  if (trim || number) {
    addHandler(el, 'blur', '$forceUpdate()')
  }

  接下来通过genAssignmentCode方法生成code，这个方法我们在分析parse时简单了解了一下，因为处理.sync修饰符时也用到了它。我们先继续往下看，最后其实就是v-model的实质，其实就是为我们提供了一个语法糖，自动为我们添加一个value的props，然后为我们添加一个@input事件，当然如果使用.lazy会是一个@change事件。
  如果我们使用.trim和.number修饰符，还会添加一个失去焦点事件，事件函数则是$forceUpdate去更新组件。我们看下面例子

new Vue({
  el: '#app',
  data: {
    value: ''
  },
  // template: `
  // <input type="text" v-model.trim="value">
  // `,
  // 等价于
  template: `
    <input type="text" :value="value" 
    @input="value=$event.target.value.trim()">
  `,
})

  实际上v-model就是下面这个模板简写的语法糖。如果有.trim修饰符时，input事件中会调用trim方法，这个时候我们来思考一下会发生什么事，首先value为空，我们输入1，这时视图上显示的'1'，然后我们输入空格，这时视图显示的'1 '，但是我们事件处理时将value修改为'1 '.trim()，也就是'1'。这时我们发现value的值实际上没有改变，所以是不会触发更新的，因为在set拦截中做了判断。但是这个时候视图显示是有空格的，如果不去更新视图，那么视图和数据就不统一了，那么这里就采用了强制更新的方式去更新视图，这样就没有问题了。
  接下来我们来看genAssignmentCode，它会传入value和valueExpression，实际上它就是返回value=valueExpression。以我们上面例子来看就是：

"value=$event.target.value.trim()"

这样的字符串。
我们来看这个方法

/**
 * Cross-platform codegen helper for generating v-model value assignment code.
 * 用于生成v-model值分配代码的跨平台codegen助手
 */
export function genAssignmentCode (
  value: string,
  assignment: string
): string {
  const res = parseModel(value)
  if (res.key === null) {
    return `${value}=${assignment}`
  } else {
    return `$set(${res.exp}, ${res.key}, ${assignment})`
  }
}

  可以看到res.key为null的话就返回了我们上面我们demo的形式，否则会返回一个$set方法的调用。
  那我们来看parseModel是怎么解析value生成res的

let len, str, chr, index, expressionPos, expressionEndPos
export function parseModel (val: string): ModelParseResult {
  // Fix https://github.com/vuejs/vue/pull/7730
  // allow v-model="obj.val " (trailing whitespace)
  val = val.trim()
  len = val.length
  // a || a[b].c || a.c
  if (val.indexOf('[') < 0 || val.lastIndexOf(']') < len - 1) {
    index = val.lastIndexOf('.')
    if (index > -1) { // a[b].c || a.c
      return {
        exp: val.slice(0, index), // a[b] || a
        key: '"' + val.slice(index + 1) + '"' // "c"
      }
    } else {
      return { // a
        exp: val, // a
        key: null
      }
    }
  }

  str = val
  index = expressionPos = expressionEndPos = 0
  // a[b] || a["b"]
  while (!eof()) { // index < len
    chr = next() // index = 1 开始取charCodeAt，每次++index
    /* istanbul ignore if */
    if (isStringStart(chr)) { // 如果是" '开始的字符串
      parseString(chr) // 找到第二个" '时返回  处理a["b"]中b
    } else if (chr === 0x5B) { // [
      parseBracket(chr) // 处理a["b"] 中"b"
    }
  }

  return {
    exp: val.slice(0, expressionPos), // a["b"]中的 a
    key: val.slice(expressionPos + 1, expressionEndPos) // a中的"b
  }
}

这个东西没什么意思，主要是对我们复杂绑定情况的处理，比如：

/**
 * - test
 * - test[key]
 * - test[test1[key]]
 * - test["a"][key]
 * - xxx.test[a[a].test1[key]]
 * - test.xxx.a["asa"][test1[key]]
 */

  如果我们绑定一个对象中的内容，就会获取到最后一个key值放到我们返回的key中，比如绑定a.b返回一个{exp: a, key: 'b'}。然后获取到key值我们就会返回一个$set方法，这样我们即使绑定了一个不存在的属性也可以添加它并将它变为响应式的。
parseString方法会处理引号中内容

function parseString (chr: number): void {
  const stringQuote = chr
  while (!eof()) {
    chr = next()
    if (chr === stringQuote) {
      break
    }
  }
}

parseBracket处理中括号中内容

function parseBracket (chr: number): void {
  let inBracket = 1
  expressionPos = index
  while (!eof()) {
    chr = next()
    if (isStringStart(chr)) {
      parseString(chr)
      continue
    }
    if (chr === 0x5B) inBracket++
    if (chr === 0x5D) inBracket--
    if (inBracket === 0) {
      expressionEndPos = index
      break
    }
  }
}

  这部分内容做个了解就可以了，没必要去详细分析了。因为了解它确实意义也不大。
  我们说非组件的model调用会返回一个true，那么这个true是什么呢？，它会返回给genDirectives方法中的needRuntime，needRuntime存在就会返回一个对象以便于运行时做一些处理，也就是genData时会给data添加一个dirs属性。在运行阶段就可以获取到这个属性去做一些处理。
