# 6.7v-model（3）

  首先来看inserted钩子，这里定义的是inserted，但是mergeVnodeHook时是合并到insert钩子中的。

  inserted (el, binding, vnode, oldVnode) {
    if (vnode.tag === 'select') {
    } else if (vnode.tag === 'textarea' || isTextInputType(el.type)) {
      el._vModifiers = binding.modifiers
      if (!binding.modifiers.lazy) {
        el.addEventListener('compositionstart', onCompositionStart)
        el.addEventListener('compositionend', onCompositionEnd)
        // Safari < 10.2 & UIWebView doesn't fire compositionend when
        // switching focus before confirming composition choice
        // this also fixes the issue where some browsers e.g. iOS Chrome
        // fires "change" instead of "input" on autocomplete.
        el.addEventListener('change', onCompositionEnd)
        /* istanbul ignore if */
        if (isIE9) {
          el.vmodel = true
        }
      }
    }
  },

  我们先不看select选择器逻辑，来看else if逻辑，如果没有设置lazy修饰符，我们会添加一个compositionstart事件监听和compositionend事件监听，我们来看onCompositionStart事件

function onCompositionStart (e) {
  e.target.composing = true
}

  也就是把e.target.composing设置为true

function onCompositionEnd (e) {
  // prevent triggering an input event for no reason
  if (!e.target.composing) return
  e.target.composing = false
  trigger(e.target, 'input')
}

  onCompositionEnd 则是将e.target.composing设置为false，但是如果它已经是false了那就什么都不需要做。否则还要执行一个trigger方法。   trigger方法如下

function trigger (el, type) {
  const e = document.createEvent('HTMLEvents')
  e.initEvent(type, true, true)
  el.dispatchEvent(e)
}

  这里会通过document.createEvent方法创建一个对象。这个方法可以创建一个指定类型的事件。其返回的对象必须先初始化并可以被传递给 element.dispatchEvent。它会接收一个type参数，也就是事件类型事件类型可能包括"UIEvents"、"MouseEvents"、"MutationEvents" 或者 "HTMLEvents"。不同类型会有不同初始化函数，HTMLEvents初始化函数为initEvent。
  也就是下面我们执行的e.initEvent(type, true, true)，第一个传入的type，也就是事件名input，第二个布尔值是决定是否冒泡，第三个是cancelable，决定是否可以取消，true则代表可以取消。
  然后通过el.dispatchEvent(e)去触发。EventTarget 的 dispatchEvent() 方法会向一个指定的事件目标派发一个 Event，并以合适的顺序（同步地）调用所有受影响的 EventListener。和经由浏览器触发，并通过事件循环异步调用事件处理程序的“原生”事件不同，dispatchEvent()会同步调用事件处理函数。在 dispatchEvent() 返回之前，所有监听该事件的事件处理程序将在代码继续前执行并返回。
  那么这段代码到底是在执行什么逻辑呢？首先它要保证你处理的是一个文本输入框。当我们输入汉字时可以认为是字母的合成，输入法输入汉字开始能被compositionstart监听到，最终确认合成时会被和compositionend监听到。
  正常来说我们输入“学习”的“学”字，首先要输入x，然后是u和e。输入x时其实就会触发@input事件，当然我们使用lazy修饰符除外。所以逻辑执行要把lazy修饰符排除在外。如果输入一个汉字就去多次触发修改的话显然是不合理的。所以这里在输入开始的时候将e.target.composing设置为true。
  这就和我们代码生成过程中这样一段代码对应上了

  if (needCompositionGuard) {
    code = `if($event.target.composing)return;${code}`
  }

  也就是说v-model绑定的input事件中会根据composing的值去判断是否修改绑定的变量，直到最后确定合成后才进行一次修改。
  但是如果我们打时没有确定合成，而是切换焦点，那么就出现问题了，所以就监听一个change事件去修复这个bug。正好它还能修复输入法自动完成时会触发change而不是input的问题。
  对于componentUpdated方法，这里的处理都是针对select标签的处理，这里我们就先不讨论了。
  接下来我们思考一个问题，为什么v-model指令没有bind这些方法呢？我们在自定义指令时会写bind函数，update函数等等。那为什么v-model指令不用呢？我认为Vue实现指令是有两种思路的，一种是在编译时去处理指令，比如我们v-for、v-if等指令，它在代码生成节点就已经生成好了代码，因为这部分指令不是针对功能上的实现，而是针对模板的渲染。而另一种是运行时实现，会对生成的dom做一些处理。
  实际上v-model也是在编译时就已经处理好了，我们可以看到这里的运行时的方法只是做了一些增强。 下面来看组件v-model的实现，先来看genComponentModel方法

export function genComponentModel (
  el: ASTElement,
  value: string,
  modifiers: ?ASTModifiers
): ?boolean {
  const { number, trim } = modifiers || {}

  const baseValueExpression = '$$v'
  let valueExpression = baseValueExpression
  if (trim) {
    valueExpression =
      `(typeof ${baseValueExpression} === 'string'` +
      `? ${baseValueExpression}.trim()` +
      `: ${baseValueExpression})`
  }
  if (number) {
    valueExpression = `_n(${valueExpression})`
  }
  const assignment = genAssignmentCode(value, valueExpression)
  el.model = {
    value: `(${value})`,
    expression: JSON.stringify(value),
    callback: `function (${baseValueExpression}) {${assignment}}`
  }
}

  这是用来处理组件v-model的方法的，定义baseValueExpression = '$$v'，然后将其赋值给valueExpression。如果trim为true则生成一个代码串。
  这段代码会判断$$v类型是否是string，如果是的话就执行trim方法，如果不是的就直接返回它本身。
  如果number为true，返回_n(${valueExpression})。
  然后同样是通过genAssignmentCode生成一个code，并保存在assignment变量中，不同的是这里会在el上挂一个model对象，有value、express、callback属性。
我们来写一个小demo

const child = {
  template: `<input :value="value" @input="input">`,
  props: ['value'],
  methods: {
    input(e) {
      this.$emit('input', e.target.value)
    }
  }
}

new Vue({
  el: '#app',
  data: {
    modelValue: ''
  },
  components: { child },
  template: `<child v-model="modelValue"></child>
  `
})

  然后返回的el.model是一个这样的对象，可以看到$$v其实就是函数的形参，所以定义了这么一个怪异的名字。

{
  value: "(modelValue)",
  expression: "\"modelValue\"",
  callback: "function ($$v) {modelValue=$$v}"
}

  执行完这一步以后，回到了model函数，model函数会直接返回false，这里返回false的话那么genDirectives方法就执行结束了，这时el有了model属性，我们回到genData中有这样一段逻辑

  // component v-model
  if (el.model) {
    data += `model:{value:${
      el.model.value
    },callback:${
      el.model.callback
    },expression:${
      el.model.expression
    }},`
  }

  其实就是把我们上面的三个属性都加入到data.model中，然后编译过程结束。编译结果为with(this){return _c('child',{model:{value:(modelValue),callback:function ($$v) {modelValue=$$v},expression:\"modelValue\"}})}
等到我们运行时创建child组件vnode时会调用createComponent方法，其中有这样一段逻辑

  // transform component v-model data into props & events
  if (isDef(data.model)) {
    transformModel(Ctor.options, data)
  }

  如果data.model存在时会执行transformModel方法

// transform component v-model info (value and callback) into
// prop and event handler respectively.
function transformModel (options, data: any) {
  const prop = (options.model && options.model.prop) || 'value'
  const event = (options.model && options.model.event) || 'input'
  ;(data.attrs || (data.attrs = {}))[prop] = data.model.value
  const on = data.on || (data.on = {})
  const existing = on[event]
  const callback = data.model.callback
  if (isDef(existing)) {
    if (
      Array.isArray(existing)
        ? existing.indexOf(callback) === -1
        : existing !== callback
    ) {
      on[event] = [callback].concat(existing)
    }
  } else {
    on[event] = callback
  }
}

  这个方法接收两个参数一个是组件定义时的options，另一个则是data。这里首先从model上获取prop和event，如果获取不到的话则默认为value和input，然后会给data.attrs添加一个prop对应属性，这里默认为value属性，使它等于model.value。
  然后就是事件处理先获取data.on中event对应事件保存在existing中，默认去找input事件。如果existing不为空，说明之前已经有input事件了，那么再去判断这个事件是否是一个数组，如果是数组并且数组中不存在我们model.callback，那么就将callback添加到数组的第一位并赋值给on.input。如果之前existing定义不是一个数组而是单独的函数，那么existing不等于model.callback时要把这两个函数都放到一个数组中并赋值给on.input。
  如果之前没有input事件，那么直接赋值on.input为callback。这是默认model的情况，实际上组件还可以自定义model绑定的prop和event。我们直接看官网的一个示例

Vue.component('base-checkbox', {
  model: {
    prop: 'checked',
    event: 'change'
  },
  props: {
    checked: Boolean
  },
  template: `
    <input
      type="checkbox"
      :checked="checked"
      @change="$emit('change', $event.target.checked)"
    >
  `
})

  这时我们可以从options中取到model.prop和model.event，而不去使用默认的value和input，以防止一些条件下的冲突。
  所以组件的v-model绑定会在render阶段将事件添加到data.on中，最后在patch时交给事件处理机制去处理。
