# 6.9 slot（上）

  对于slot的parse过程我们之前在5.7小节简单分析过了。下面我们直接来看代码生成过程，先写一个小demo

const child = {
  template: `<div>
    我是<slot>1</slot>
  </div>`
}

new Vue({
  el: '#app',
  components: { child },
  template: `<child></child>`
})

  在代码生成阶段解析到el.tag === slot时会执行genSlot方法

function genSlot (el: ASTElement, state: CodegenState): string {
  const slotName = el.slotName || '"default"'
  const children = genChildren(el, state)
  let res = `_t(${slotName}${children ? `,function(){return ${children}}` : ''}`
  const attrs = el.attrs || el.dynamicAttrs
    ? genProps((el.attrs || []).concat(el.dynamicAttrs || []).map(attr => ({
        // slot props are camelized
        name: camelize(attr.name),
        value: attr.value,
        dynamic: attr.dynamic
      })))
    : null
  const bind = el.attrsMap['v-bind']
  if ((attrs || bind) && !children) {
    res += `,null`
  }
  if (attrs) {
    res += `,${attrs}`
  }
  if (bind) {
    res += `${attrs ? '' : ',null'},${bind}`
  }
  return res + ')'
}

  在parse过程中如果解析到slot标签name属性，会将其保存在el.slotName中，这里先取到slotName，如果没有定义name，就将其赋值为"default"，接下来执行genChildren方法处理slot中的内容。也就是我们上面slot中写的文本"1"。我们知道插槽是允许添加默认内容的，假如我们没有在父组件<child>标签中写入任何内容，则会使用插槽的默认内容。所以genChildren就是去处理slot中间文本的。 然后res会被定义为:
"_t(\"default\",function(){return [_v(\"1\")]}"
  我们先继续往下看，最后再来看_t是什么，判断el.atrts或el.dynamicAttrs存在的话就将其合并到一个数组中并通过map方法返回name、value、dynamic属性组成的数组。这里name会处理为驼峰形式。然后将这个数组交给genProps方法去处理，最后结果赋值给attrs。
  接下来看有没有v-bind属性，如果bind和attrs至少有一个不为空并且children不存在的话就给res拼接一个',null'。
  如果attrs存在，也是将它拼接到res，对于bind也是同样处理，其实这些内容都作为_t函数的参数传入。最后返回res + )把_t函数表达式补充完整。
  接下来我们分析运行时的逻辑，我们父组件最后生成的代码为"_c('child',[_v("子")])"，可以看到这是非常简单的。等到运行时我们执行render函数时会执行这段代码，这时会先执行_v("子")生成子vnode，然后才会执行_c函数，也就是createElement方法。然后会发现这里的tag为'child'，也就是一个组件标签，那么就会执行创建组件vnode的方法createComponent，这个方法最后生成vnode执行

  const vnode = new VNode(
    `vue-component-${Ctor.cid}${name ? `-${name}` : ''}`,
    data, undefined, undefined, undefined, context,
    { Ctor, propsData, listeners, tag, children },
    asyncFactory
  )

  这个东西我们见过很多次了，可以说它是非常重要的。这里传入的children，其实这里就是组件vnode的子vnode。我们正常组件中间是没有内容的。如果还有子元素的话那必然就是我们插槽了。这里的children其实就是一个只有一个文本vnode的数组。创建vnode时这整个配置对象会赋值给vnode.componentOptions。
  然后再patch过程中会初始化子组件构造器，也就是执行下面这个方法

export function createComponentInstanceForVnode (
  // we know it's MountedComponentVNode but flow doesn't
  vnode: any,
  // activeInstance in lifecycle state
  parent: any

    _isComponent: true, // 记住这个标识，在Vue初始化逻辑中就对它进行了判断

    parent

  return (node.isComment && !node.asyncFactory) || node.text === ' '

  也就是renderSlot方法

export function renderSlot (
  name: string,
  fallbackRender: ?((() => Array<VNode>) | Array<VNode>),
  props: ?Object,
  bindObject: ?Object
): ?Array<VNode> {
  const scopedSlotFn = this.$scopedSlots[name]
  let nodes
  if (scopedSlotFn) {
    // scoped slot
    props = props || {}
    if (bindObject) {
      if (process.env.NODE_ENV !== 'production' && !isObject(bindObject)) {
        warn('slot v-bind without argument expects an Object', this)
      }
      props = extend(extend({}, bindObject), props)
    }
    nodes =
      scopedSlotFn(props) ||
      (typeof fallbackRender === 'function' ? fallbackRender() : fallbackRender)
  } else {
    nodes =
      this.$slots[name] ||
      (typeof fallbackRender === 'function' ? fallbackRender() : fallbackRender)
  }

  const target = props && props.slot
  if (target) {
    return this.$createElement('template', { slot: target }, nodes)
  } else {
    return nodes
  }
}

  这里会先根据name从this.$scopedSlots中拿到scopedSlotFn，这里name取值为default，拿到的就是我们上面proxyNormalSlot返回的函数，这个函数其实就是会返回一个默认的插槽的渲染函数，如果没有默认插槽就会返回null。
  接下来没有props会初始化props，也就是我们在slot标签上传入的props属性，我们会使用它来做作用于插槽的传值。当然这里我们没有添加属性，但是不影响我们分析。接下来获取bindObject，这个则是我们去v-bind传多个props配置才会有值。如果bindObject不是一个object的话就会报一个警告，接下来会把props和bindObject合并。
  接下来会将nodes赋值为scopedSlotFn执行结果，也就是会返回我们default中的children数组，如果没有定义插槽内容的话，那就会找fallbackRender，如果存在则会执行它去获取默认插槽内容，当然可能默认插槽也没有定义，那么fallbackRender则是为null。把null直接赋值给nodes就好了。
  至于没有scopedSlotFn，那就是另一种情况了，这个是normalizeScopedSlots执行不同分支才会有的问题，这里我们只需要知道它会做什么操作即可。其实和上面是一致的，因为对于目前的case来说scopedSlotFn本质也是从this.$slot去获取name对应children。
  如果发现props上还有slot属性，那么会调用$createElement去创建template的vnode，并传递slot属性，和nodes子vnode，否则就直接返回nodes。
  对于子组件的render函数其实是这样的_c('div',[_v("\n 我是"),_t("default",function(){return [_v("1")]})],2)。上述过程是执行完了_t()，然后就会执行_c函数去生成div的vnode了。其实我们_t函数的作用就是取到父组件中定义的插槽内容的vnode然后放到子组件中，这样patch过程中就会插入到子组件了。
  到这里我们的普通插槽其实就分析完成了。但是我们知道插槽还有另外两种：具名插槽和作用域插槽，那么下个小节我们就来分析它们吧！
