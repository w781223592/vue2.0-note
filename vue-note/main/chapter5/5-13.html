# 5.11 generate代码生成（3）

  这小节我们先来看一个简单的例子

new Vue({
  el: '#app',
  template: `<div :key="1" a="2":b="3" ref="d" v-if="true">
    <input type="text" :value="1"/>
  </div>`
})

  我们来看看这段代码会执行什么逻辑：首先是执行genElement方法，然后会进入el.if分支判断从而执行genIf方法

export function genIf (
  el: any,
  state: CodegenState,
  altGen?: Function,
  altEmpty?: string
): string {
  el.ifProcessed = true // avoid recursion
  return genIfConditions(el.ifConditions.slice(), state, altGen, altEmpty)
}

  将el.ifProcessed置为true，然后返回genIfConditions函数执行结果。

function genIfConditions (
  conditions: ASTIfConditions,
  state: CodegenState,
  altGen?: Function,
  altEmpty?: string
): string {
  if (!conditions.length) {
    return altEmpty || '_e()'
  }
  const condition = conditions.shift()
  if (condition.exp) {
    return `(${condition.exp})?${
      genTernaryExp(condition.block)
    }:${
      genIfConditions(conditions, state, altGen, altEmpty)
    }`
  } else {
    return `${genTernaryExp(condition.block)}`
  }
}

  这里只传入了两个参数，conditions是el.ifConditions浅拷贝而来的。如果其长度为0就返回一个_e()

  target._e = createEmptyVNode

  它是用来创建一个空的vnode的方法。
  如果conditions长度不为0，然后取到conditions中第一个元素condition，并让其第一个元素出栈。
  然后判断condition的表达式是否存在，存在的话就返回一个三元表达式的字符串，我们来看genTernaryExp是什么。

  // v-if with v-once should generate code like (a)?_m(0):_m(1)
  function genTernaryExp (el) {
    return altGen
      ? altGen(el, state)
      : el.once
        ? genOnce(el, state)
        : genElement(el, state)
  }

  genTernaryExp方法其实就是定义在genIfConditions函数中的，altGen这里是都不存在的，我们直接看el.once后面逻辑就行了，这个方法其实我们可以猜到它是生成一个当表达式exp判断成立时的执行代码。显然这里el.once是不存在的，所以再次调用genElement方法去生成code。然后如果这个exp表达式不成立生成的代码则是通过递归调用genIfConditions生成的。
  这里逻辑可能是有些绕的，但是我们来梳理它一下，如果说conditions有三项内容，分别是v-if、v-else-if、v-else的内容。我们要生成代码当然需要判断v-if的表达式是否为true，如果是true的话我们就要渲染v-if的内容，所以生成的这个三元表达式是如果condition.exp成立就执行v-if的渲染函数，而genTernaryExp中执行的genElement就是生成v-if渲染函数的code。这里我们始终得记住我们是在生成代码，而不是执行代码。
  所以我们不知道condition.exp是否是成立的，还需要生成其不成立的代码，那么假设第一个condition.exp不成立，那么我们还要再次执行genIfConditions，这个时候conditions中的v-if的内容已经出栈，那么就是开始处理v-else-if的内容，就这样把条件判断下的所有代码都要生成。最后处理v-else时是exp不存在的情况，所以直是直接进入else分支的。最后我们简单模拟一下生成的代码：

`v-if 
  ? _c(v-if) 
  : v-else-if
    ? _c(v-else-if)
    : _c(v-else)
`

  这并非真实结果，但我想它可以很直观的反应最终的结果。
  这就是条件控制指令的过程了，但是我们这里执行genElement时会执行genData方法，这里因为我们又添加key属性，ref属性，还有a属性，所以会执行一些不一样的分支内容

  // key
  if (el.key) {
    data += `key:${el.key},`
  }
  // ref
  if (el.ref) {
    data += `ref:${el.ref},`
  }
  // attributes
  if (el.attrs) {
    data += `attrs:${genProps(el.attrs)},`
  }
  // DOM props
  if (el.props) {
    data += `domProps:${genProps(el.props)},`
  }

  而我们input标签使用了:value会被当成原生dom属性处理，也就是input.props是有值的，这里我们就一起分析了，我们这里可以看一下genProps的内容

function genProps (props: Array<ASTAttr>): string {
  let staticProps = ``
  let dynamicProps = ``
  for (let i = 0; i < props.length; i++) {
    const prop = props[i]
    const value = __WEEX__
      ? generateValue(prop.value)
      : transformSpecialNewlines(prop.value)
    if (prop.dynamic) {
      dynamicProps += `${prop.name},${value},`
    } else {
      staticProps += `"${prop.name}":${value},`
    }
  }
  staticProps = `{${staticProps.slice(0, -1)}}`
  if (dynamicProps) {
    return `_d(${staticProps},[${dynamicProps.slice(0, -1)}])`
  } else {
    return staticProps
  }
}

  这个函数比较简单，主要就是将动态参数和静态参数分别拼接为key,value,的字符串保存在各自的变量中，如果存在动态参数则用一个_d方法包裹，并将staticProps包裹为对象作为第一个参数传入，第二个参数则是dynamicProps处理成一个数组字符串传入。如果没有动态参数就直接返回静态参数的字符串staticProps。
  我们来看_d方法

  target._d = bindDynamicKeys

  我们来看bindDynamicKeys方法

// helper to process dynamic keys for dynamic arguments in v-bind and v-on.
// For example, the following template:
// <div id="app" :[key]="value">
// compiles to the following:
// _c('div', { attrs: bindDynamicKeys({ "id": "app" }, [key, value]) })

import { warn } from 'core/util/debug'

export function bindDynamicKeys (baseObj: Object, values: Array<any>): Object {
  for (let i = 0; i < values.length; i += 2) {
    const key = values[i]
    if (typeof key === 'string' && key) {
      baseObj[values[i]] = values[i + 1]
    } else if (process.env.NODE_ENV !== 'production' && key !== '' && key !== null) {
      // null is a special value for explicitly removing a binding
      warn(
        `Invalid value for dynamic directive argument (expected string or null): ${key}`,
        this
      )
    }
  }
  return baseObj
}

  这个方法实际上没有什么特别之处，就是这些动态key的属性添加到attrs中，这里偶数索引都是key值，i + 1则是其value值。注意这里[key, value]都没有加""，说明它们都是变量。可能你会有疑问，最终都是把属性添加到attrs中，这和静态参数有什么区别呢？其实确实是没有区别的，毕竟都是属性嘛！但是调用时机是有区别的，动态参数key的处理是在运行时才会执行，而静态参数key则是编译时就处理好了。也就是说我们执行这个render函数时才会往里面添加key，这样才能保证key是动态的。
  再来说对el.props的处理，对于input标签的:value我们要当成dom 属性来处理，它同样是调用genProps方法，但是最后是添加到domProps中的，而不是在attrs中。

  到这里我们对v-if的处理就分析完毕了，接下来我们来分析v-once指令。我们先来看下面一个例子

new Vue({
  el: '#app',
  template: `<div v-if="true" v-once>
    我是1
  </div>`
})

  这里我们绑定了v-once指令。那么执行generate时会执行genElement，我们同时绑定了v-once和v-if，但是在genElement中el.once的处理优先级要比el.if高，所以这里会进入genOnce方法

// v-once
function genOnce (el: ASTElement, state: CodegenState): string {
  el.onceProcessed = true
  if (el.if && !el.ifProcessed) {
    return genIf(el, state)
  } else if (el.staticInFor) {
	...
  } else {
    return genStatic(el, state)
  }
}

  这里会发现el.if存在那么就会执行genIf方法并返回，这时el.onceProcessed被设置为true。在genIf执行genElement方法也不会再度进入genOnce逻辑了，所以这该怎么办呢？那就只能在genIf中执行了genOnce了，所以我们genIf中有这样一段逻辑

  function genTernaryExp (el) {
    return altGen
      ? altGen(el, state)
      : el.once
        ? genOnce(el, state)
        : genElement(el, state)
  }

  处理完当前v-if节点后就会重新判断有无el.once，然后重新执行genOnce方法，这样就会走到genOnce中的else逻辑，去执行genStatic把它处理成一个静态的节点。所以这里genOnce更像是拦截了genIf本身的生成代码过程，而强制让它通过genOnce生成一个静态的节点。
  其实v-once这里还处理了一些v-for的逻辑，也就是else if(el.staticInFor)中的逻辑

  let key = ''
  let parent = el.parent
  while (parent) {
    if (parent.for) {
      key = parent.key
      break
    }
    parent = parent.parent
  }
  if (!key) {
    process.env.NODE_ENV !== 'production' && state.warn(
      `v-once can only be used inside v-for that is keyed. `,
      el.rawAttrsMap['v-once']
    )
    return genElement(el, state)
  }
  return `_o(${genElement(el, state)},${state.onceId++},${key})`

  先定义key和parent，parent为父元素，如果parent.for存在就使key等于parent的key并跳出循环，否则就继续往上找父元素的父元素，直到parent.key存在或者parent不存在为止，如果获取到的key还是空，那么就要报一个警告说v-once只能用在定义key的v-for中。
  最后会返回一个_o函数的调用，其定义如下

  target._o = markOnce

markOnce方法如下

export function markOnce (
  tree: VNode | Array<VNode>,
  index: number,
  key: string
) {
  markStatic(tree, `__once__${index}${key ? `_${key}` : ``}`, true)
  return tree
}

  实际上就是调用markStatic去标记v-once的特殊标识。其实这些方法都是运行时调用的，这时候会对生成的vnode打上这些指令特有的标记方便进一步去处理。
