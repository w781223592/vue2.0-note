  for循环有很多的if else判断，可以说是比较复杂的。首先把属性名称保存在name和rawName中。把属性值保存在value中。
  接下来if判断是否是一个指令，如果是指令就执行if逻辑，否则就执行else逻辑。我们先看是指令的情况，首先将el.hasBindings设置为true。然后获取修饰符modifiers，这里是通过parseModifiers方法获取的，并且将name的指令前缀去掉作为参数传入。

function parseModifiers (name: string): Object | void {
  const match = name.match(modifierRE)
  if (match) {
    const ret = {}
    match.forEach(m => { ret[m.slice(1)] = true })
    return ret
  }
}

  通过modifierRE正则来匹配name，比如对click.stop指令将会匹配其.stop，接下来就把匹配到的修饰符都放到ret对象中，这里是将匹配结果截掉第一个位的'.'，也就是生成一个对象：

ret = {
  stop: true
}

  最后就将ret对象保存到了modifiers中，接下来是.prop修饰符逻辑

  // support .foo shorthand syntax for the .prop modifier
  if (process.env.VBIND_PROP_SHORTHAND && propBindRE.test(name)) {
    (modifiers || (modifiers = {})).prop = true
    name = `.` + name.slice(1).replace(modifierRE, '')
  } else if (modifiers) {
    name = name.replace(modifierRE, '')
  }

  若name通过propBindRE正则检查，如果检查通过则将modifiers.prop置为true。这里是指简写的prop修饰符，比如将foo.prop写为.foo然后就可以通过这段代码处理，最后处理的name结果为.foo。
  如果不是.prop修饰符简写的话就直接将使name的修饰符去掉并赋值给name。
  接下来的是对v-bind的判断，if(bindRE.test(name))，这部分内容比较多我们拆开分析

    name = name.replace(bindRE, '')
    value = parseFilters(value)
    isDynamic = dynamicArgRE.test(name)
    if (isDynamic) {
      name = name.slice(1, -1)
    }
    if (
      process.env.NODE_ENV !== 'production' &&
      value.trim().length === 0
    ) {
      warn(
        `The value for a v-bind expression cannot be empty. Found in "v-bind:${name}"`
      )

  如果是v-bind指令，就先将其前缀去掉，然后再将绑定值value中可能存在的过滤器处理，并将处理结果返回给value。然后判断是否是动态参数绑定，如v-bind:[a]形式。如果是动态参数绑定则截取其第一位和最后一位中间的内容，也就是这里的[a]中的a。如果发现绑定值是一个空值则要在非生产环境报一个警告。

  如果修饰符存在，则对其进一步处理，首先如果有prop修饰符并且不是一个动态参数，然后要对属性名做一个连字符转驼峰处理。如果name完全等于innerHtml时要转化为innerHTML。
  如果有camel修饰符，同样是要转驼峰处理。
  如果有sync修饰符，会执行genAssignmentCode方法

// Cross-platform codegen helper for generating v-model value assignment code.
// 用于生成v-model值分配代码的跨平台codegen助手
export function genAssignmentCode (
  value: string,
  assignment: string
): string {
  const res = parseModel(value)
  if (res.key === null) {
    return `${value}=${assignment}`
  } else {
    return `$set(${res.exp}, ${res.key}, ${assignment})`
  }
}

  通过parseModel方法得到res，如果res.key值完全等于null就直接return ${value}=${assignment}。否则为$set(${res.exp},${res.key},${assignment})。
  这个方法也会在v-model指令时调用，实际上.sync就是一种v-model的实现。在vue3中其实已经都用v-model代替。
  至于后面判断是非动态参数然后调用addHandler方法多添加一个事件监听。这个事件名是update: + 属性名的形式，相信我们都这样写过。然后如果name的连字符形式和它的驼峰形式不一样的话，就再监听一个连字符属性的命事件。
  如果是动态参数的话，则同样是执行addHandler方法，但是最后要传一个dynamic参数为true。对于这部分我们了解到这里就可以了，我们等之后章节学习v-on、v-model等指令时再详细分析这些内容。
  再往下看

  if ((modifiers && modifiers.prop) || (
    !el.component && platformMustUseProp(el.tag, el.attrsMap.type, name)
  )) {
    addProp(el, name, value, list[i], isDynamic)
  } else {
    addAttr(el, name, value, list[i], isDynamic)
  }

  如果当面添加了modiriers.prop为true，或者!el.component并且是DOM元生属性的话则执行addProp方法将其添加到el.props。如果不满足上述条件就调用addAttr处理其他属性。
  分析完v-bind指令我们来看else if逻辑，也就是对v-on的处理

  这部分处理比较简单，先将name事件前缀去掉，然后再检验是否是动态命名参数，如果是的话还是同样的处理。最后通过addHandler处理事件。
  接下来还要解析其他指令，如：v-text、v-html、v-show、v-cloak、v-model，还可能有我们自己定义的指令。

    isDynamic = false

        arg = arg.slice(1, -1)
        isDynamic = true

  处理非常简单，先将指令前缀v-去掉，然后取指令参数，再将isDynamic设置为false。判断是否有arg参数，如果有arg，就先截取指令的名称，也就是v-后面':'前面的字符串。比如v-model的model，然后同样判断动态参数并处理。
  接下来往el.directives中添加指令对象。如果是v-model的话会在非生产环境调用checkForAliasModel方法。这个方法主要检查v-model绑定的值是不是v-for循环遍历出的item，如下：

function checkForAliasModel (el, value) {
  let _el = el
  while (_el) {
    if (_el.for && _el.alias === value) {
      warn(
        `<${el.tag} v-model="${value}">: ` +
        `You are binding v-model directly to a v-for iteration alias. ` +
        `This will not be able to modify the v-for source array because ` +
        `writing to the alias is like modifying a function local variable. ` +
        `Consider using an array of objects and use v-model on an object property instead.`,
        el.rawAttrsMap['v-model']
      )
    }
    _el = _el.parent
  }
}

  如果使用v-model的标签绑定了v-for指令或者其父元素绑定了v-for指令，并且el.alias === value，就会报一个警告说：您正在将v-model直接绑定到v-for迭代别名这将无法修改v-for源数组，因为写入别名就像修改函数局部变量考虑使用对象数组，并在对象属性上使用v-model。
  也就是说下面这种写法是不行的

   <div v-for="item in list">
    <input type="text" v-model="item">
   </div>
但是可以写成
   <div v-for="item in list">
    <input type="text" v-model="item.value">
   </div>

  但是我们还是一些文本属性的，比如id、width等属性，至于style和class我们是在transforms中处理并做了增强。下面我们就来看看这些属性是怎么处理的：

  文本属性其实就是静态文本，也就是属性值是字符串形式，所以这里使用JSON的方法格式化后传入addAttr方法中。接下来是对muted属性的判断，实际上我们addAttr会将属性对象添加到el.attrs数组中，生成真实dom时通过setAttribute方法去将属性添加到真实DOM上。但是在火狐浏览器（为什么总是你？）中无法通过setAttribute方法为video添加muted属性。所以处理方法也比较简单，就是调用addProp方法先将其添加到el.props中，对于props中属性是通过真实dom对象直接添加的，也就是直接videoElement.muted=true。
  最后我们来看上面这个非生产环境的警告，先通过parseText方法去解析value，将结果保存在res中，如果res存在的话说明我们写了

  <div id="{{ val }}">

这样的代码。这是不对的应该用v-bind去代替。可见parseText就是去解析插值字符串的。因为parseText方法主要是在解析文本时会用到，所以我们可以等到那个时候再去分析它。
  到这里我们就将processAttrs方法都分析完毕了。那么的processElement函数也就分析完了，我们再回到closeElement方法中

  // tree management
  if (!stack.length && element !== root) {
    // allow root elements with v-if, v-else-if and v-else
    if (root.if && (element.elseif || element.else)) {
      if (process.env.NODE_ENV !== 'production') {
        checkRootConstraints(element)
      }
      addIfCondition(root, {
        exp: element.elseif,

      warnOnce(
        `Component template should contain exactly one root element. ` +
        `If you are using v-if on multiple elements, ` +
        `use v-else-if to chain them instead.`,
        { start: element.start }

  这里如果stack.length为0，并且不是根节点时，这是什么情况呢？我们知道调用目前closeElement的地方有两处的，一个start钩子中遇到单标签，另一个是end钩子中。调用start钩子时，如果这时是一个双标签，就会往stack添加一个元素，而执行end时会让stack弹出栈顶元素，然后执行closeElement。
  我们之前说过解析到的第一个开始标签就认为是根标签，它的元素信息会交给root保存，而这里stack.length等于0并且不是根元素的话说明root标签被闭合了。但是现在仍然解析到了，说明存在了多个根标签，这本来是不被允许的，但是在一种情况下它应该被允许发生，那就是root元素上有v-if指令，并且目前解析到的第二个根标签绑定了v-else-if或v-else，但是这个情况仍要通过checkRootConstraints检查才是被允许的，否则要报一个警告。
  接下来就是处理v-else-if指令和v-else指令了。同样是调用addIfCondition方法去处理，但是这里传入的第一个参数是root而不是element，而exp传入的是element.elseif。那么v-else就不管了吗？其实不是的，是因为我们在写v-else是不需要添加属性值的，其实得到的就是undefined。而我们写v-else就不可能有v-else-if了，所以这时候element.elseif值也是undefined，所以就只取elseif值就够了。所以v-if系列指令是放在一起管理的。

  <div v-if="a"></div>
  <p v-else-if="b"></p>
  <span v-else></span>

  再往下closeElement方法执行了这段逻辑，如果当前父元素存在并且当前元素不是被禁用的然后就执行if内逻辑。
  如果元素绑定了v-else-if或者v-else指令时执行processIfConditions方法，我们来看processIfConditions方法做了什么事

function processIfConditions (el, parent) {
  const prev = findPrevElement(parent.children)
  if (prev && prev.if) {
    addIfCondition(prev, {
      exp: el.elseif,
      block: el
    })
  } else if (process.env.NODE_ENV !== 'production') {
    warn(
      `v-${el.elseif ? ('else-if="' + el.elseif + '"') : 'else'} ` +
      `used on element <${el.tag}> without corresponding v-if.`,
      el.rawAttrsMap[el.elseif ? 'v-else-if' : 'v-else']
    )
  }
}

  首先定义了prev，它是执行findPrevElement方法得到的，根据名称我们可以知道它是寻找前一个元素。我们先继续往下看。
  如果prev.if存在则调用addIfCondition方法，其实和上面处理root节点的v-if指令是一样的，这里就不赘述了。这里其实是要判断前一个元素有没有绑定了v-if指令，因为v-else-if或者v-else只能紧跟在绑定v-if指令元素后面。
  那我们来看findPrevElement

function findPrevElement (children: Array<any>): ASTElement | void {
  let i = children.length
  while (i--) {
    if (children[i].type === 1) {
      return children[i]
    } else {
      if (process.env.NODE_ENV !== 'production' && children[i].text !== ' ') {
        warn(
          `text "${children[i].text.trim()}" between v-if and v-else(-if) ` +
          `will be ignored.`,
          children[i]
        )
      }
      children.pop()
    }
  }
}

  这里接收的是父元素的children数组，其实就是包括当前元素在内的同级元素。就是简单通过循环去找，这里其实找到children数组最后一位就是当前元素的前一个兄弟元素了，只要把它返回就行了，为什么是这样就得从children数组的维护说起了，接下来我们很快就会讲到，所以这里先留一个悬念。
  这里还有一点需要注意的是前一个节点的type必须是1，也就是必须是个元素节点，而不能是其他类型，比如type为3的文本节点：

  <div v-if="a"></div>中间文本
  <p v-else="b"></p>

  这样v-else绑定的节点前面有一段文本，但是再往前可以取到绑定v-if的元素，这时vue也会做一些处理，自动舍弃中间文本并报一个警告。
  如果没有绑定v-else-if或者v-else，然后要判断element.slotScope是否存在，如果存在的话获取插槽名字，然后在currentParent.scopedSlots对象中以name为键名保存。
  然接下来将element添加到父级的children数组中，然后将element.parent赋值为currentParent，这里就是建立了一个父子关系。
  接下来就要把作用域插槽的元素从element.children过滤掉，这部分逻辑有点意思。看起来好像上面刚修改了children数组下面就删除了，完全可以在上面判断slotScope时就过滤掉slotScope存在的元素。
  其实不是这样的，首先我们解决一下之前埋的坑，那就是children的维护问题，它是在这里被添加属性的。也就是解析到父元素开始标签后会解析其子元素标签，每当解析到子元素标签闭合时调用closeElement方法，然后会将子元素添加到children中，这个是顺序添加的。所以解析到带有v-if元素后会将其添加到children中。如果下一个元素是带有v-else指令的，就直接找children数组中最后一位就是前一个带有v-if的元素了，当然如果中间插入其他内容就会报警告了，这就解决了我们上面遗留问题。
  那么接下来再讨论这里的问题，这其实还是与v-if处理有关的，如果我们不将作用插槽的内容添加到children中的话就会发v-if和v-else元素直接插入作用域插槽的元素而被忽略了，这是不允许的，应该去报一个警告。所以这里处理的办法是等子元素都解析完，解析到父元素的结束标签时将其children中含有slotScope元素过滤掉。
  我们继续往下分析

  // remove trailing whitespace node again
  trimEndingWhitespace(element)

  // check pre state
  if (element.pre) {
    inVPre = false
  }
  if (platformIsPreTag(element.tag)) {
    inPre = false
  }
  // apply post-transforms
  for (let i = 0; i < postTransforms.length; i++) {
    postTransforms[i](element, options)
  }

  然后再次调用trimEndingWhitespace方法去删除尾部空格。
  然后接下来要重置一些变量，比如绑定了v-pre的标签，解析到开始标签会将inVPre设置为true，这里解析到其结束标签后就要将其重置为false。同样的inPre也要恢复为false。对于postTransform方法我们在web平台是没有的，这是值得庆幸的。
