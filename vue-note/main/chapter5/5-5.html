# 5.5 parse之生成AST（1）

  上一小节我们分析了解析模板方法parseHTML，我们了解到了template字符串是怎被一步步解析和处理的，这小节我们就来进一步分析这些标签、属性、文本等被解析后最终做了怎样的处理。
  相信我们都知道这些标签、属性、内容最后被解析成了AST，也就是我们的抽象语法树，AST到底是什么东西呢？它是抽象的我们不好去描述，那我们就把它来具体化相信就十分清楚了。
  在parse/index.js中有这样一段代码

export function createASTElement (
  tag: string,
  attrs: Array<ASTAttr>,
  parent: ASTElement | void
): ASTElement {
  return {
    type: 1,
    tag,
    attrsList: attrs,
    attrsMap: makeAttrsMap(attrs),
    rawAttrsMap: {},
    parent,
    children: []
  }
}

  这个方法返回的对象虽然不是最终的AST，但已经具有AST雏形了，比如有如下模板：

  <div>
    <span>hello</span>
  </div>

  它会生成一个如下AST模板：

{
  type: 1,
  tag: 'div',
  attrsList: [],
  attrsMap: {},
  rawAttrsMap: {},
  parent: null,
  children: [{
    type: 1,
    tag: 'span',
    attrsList: [],
    attrsMap: {},
    rawAttrsMap: {},
    parent: div,
    children: [{
      type: 3,
      text: 'hello'
    }]
  }]
}

  可以看到AST其实就是对应我们dom树，tag就是我们的标签名，type为1则代表我们元素节点，type为3则代表文本节点。这个和我们nodeType其实是一致的。   在分析parse方法时我们还有很多问题需要解决，那就是文件开头定义了大量正则常量，这些都是我们需要事先准备的，这样在分析parse过程中就不必穿插这些内容去转移我们的注意力了。parse的准备工作我们会在compile文档中去分析，我们就不在正文中呈现了。这部分代码如下：

  2. 元素之间的包括换行在内的多个空格会被移除
  3. 文本结点之间可被压缩的空格都会被压缩成为一个空格

  接下来的这段话则是为了解决ie浏览器的一个bug

  <svg xmlns:feature="http://www.xxx"></svg>
  <!-- 被渲染为： -->
  <svg xmlns:NS1="" NS1:xmlns:feature="http://www.xxx"></svg>

  可以看到会多一个xmlns:NS1=""和NS1:，所以要把它去掉才行，解决方案也很简单

const ieNSBug = /^xmlns:NS\d+/
const ieNSPrefix = /^NS\d+:/

/* istanbul ignore next */
function guardIESVGBug (attrs) {
  const res = []
  for (let i = 0; i < attrs.length; i++) {
    const attr = attrs[i]
    if (!ieNSBug.test(attr.name)) {
      attr.name = attr.name.replace(ieNSPrefix, '')
      res.push(attr)
    }
  }
  return res
}

  遍历attrs数组然后检查属性名中有没有满足isNSBug正则的字符串，有的话就过滤掉，attrs是这个样子

]

  这里又定义一个正则ieNSprefix，它就能匹配NS1:xmlns:feature前面的NS1:，然后把它删除。最后就处理成功我们正常形式：

  我们来看看makeAttrsMap的实现

function makeAttrsMap (attrs: Array<Object>): Object {
  const map = {}
  for (let i = 0, l = attrs.length; i < l; i++) {
    if (
      process.env.NODE_ENV !== 'production' &&
      map[attrs[i].name] && !isIE && !isEdge
    ) {
      warn('duplicate attribute: ' + attrs[i].name, attrs[i])
    }
    map[attrs[i].name] = attrs[i].value
  }
  return map
}

  其实就是把属性处理成一个对象name：value形式，比如说a="1"会处理为：

{
  a: 1
}

  这是一个判断，用来判断是否是被禁止的标签，因为template应该只负责把状态映射到ui。而不应该带有副作用。比如你是一个script标签，那么极可能引起副作用。当然服务器渲染不在此列。我们来看isForbiddenTag

function isForbiddenTag (el): boolean {
  return (
    el.tag === 'style' ||
    (el.tag === 'script' && (
      !el.attrsMap.type ||
      el.attrsMap.type === 'text/javascript'
    ))
  )
}
