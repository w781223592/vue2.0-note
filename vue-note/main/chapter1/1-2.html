# 1.2 Vue入口文件

Vue入口文件在src/platforms目录下，以entry-runtime-with-compiler.js为例，在compiler版本下vue重写了$mount方法并保存了原有的$mount，新增编译逻辑，最后导出Vue

import config from 'core/config'
import { warn, cached } from 'core/util/index'
import { mark, measure } from 'core/util/perf'

import Vue from './runtime/index'
import { query } from './util/index'
import { compileToFunctions } from './compiler/index'
import { shouldDecodeNewlines, shouldDecodeNewlinesForHref } from './util/compat'

从上面代码可以看出entry-runtime-with-compiler.js中导入了大量的内容，而我们的核心的Vue来自runtime/index，runtime/index则是最开始定义$mount的地方

import Vue from 'core/index'
import { mountComponent } from 'core/instance/lifecycle'
import { query /* ...省略若干 */} from 'web/util/index'

Vue.prototype.$mount = function (
  el?: string | Element,
  hydrating?: boolean
): Component {
  el = el && inBrowser ? query(el) : undefined
  return mountComponent(this, el, hydrating)
}

其定义十分简单，判断el存在且在浏览器环境，最后return mountComponent()方法， 然后我们看到关键代码import Vue from 'core/index' 打开core/index文件，又发现

import Vue from './instance/index'
import { initGlobalAPI } from './global-api/index'

initGlobalAPI(Vue)

Vue引自instance/index，core/index中初始化了全局API，并对服务端渲染做了扩展。

最后打开instance/index我们便发现Vue的定义

import { initMixin } from './init'
import { stateMixin } from './state'
import { renderMixin } from './render'
import { eventsMixin } from './events'
import { lifecycleMixin } from './lifecycle'
import { warn } from '../util/index'

function Vue (options) {
  if (process.env.NODE_ENV !== 'production' &&
    !(this instanceof Vue)
  ) {  
warn('Vue is a constructor and should be called with the `new` keyword')
  }
  // init.js中原型挂载_init方法, this指向当前Vue对象
  this._init(options)
}
initMixin(Vue)
stateMixin(Vue)
eventsMixin(Vue)
lifecycleMixin(Vue)
renderMixin(Vue)
export default Vue

难以置信，我们层层深入找到的Vue定义居然如此简单，其实现仅仅是定义一个Vue构造函数，接收一个options参数，执行this._init(options)方法，之后执行一系列Mixin混入，这里使用构造函数来实现Vue的类，因为Vue是一个函数所以可能被直接调用而不是用new去调用，所以Vue在开发环境使用Vue()调用函数会被warn，那为什么不用class去封装Vue类而要多此一举使用构造函数并对它的调用方式做判断呢? 其实Vue是一个复杂且庞大的类，我们之所以看到这个类十分简单，是因为使用构造函数方式去定义使得Vue可以在原型上扩展许多方法，这样可以将Vue的实现按照不同功能拆分为不同模块中，而不是写在一个模块中。这是class所不具备的，这种编程思路可以方便代码的管理和维护。
