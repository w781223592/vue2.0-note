# 3.3合并配置

  我们上一小节讲解了Vue组件渲染过程，到这里我们差不多已经理清Vue整体脉络，接下来的学习我们将会变的简单起来，因为我们之后分析内容会是一些穿插在主干中的一些细致末节，这些小的枝干我们理解起来会轻松很多。
  我们在init时候分两种情况进行选项的合并：一种是我们手动调用new Vue去实例化组件，另一种则是代码执行过程中自动调用new Vue去实例化组件。这表现在_init方法中的这样一段逻辑

  if (options && options._isComponent) {
    // optimize internal component instantiation 优化内部组件实例化
    // since dynamic options merging is pretty slow, and none of the 因为动态选项合并非常慢，而且
    // internal component options needs special treatment. 内部组件选项需要特殊处理
    initInternalComponent(vm, options)
  } else {
    vm.$options = mergeOptions(
      resolveConstructorOptions(vm.constructor),
      options || {},
      vm
    )
  }

我们都知道这段逻辑就是做了选项的合并，我们是不是曾认为if逻辑是组件选项的合并，而else逻辑为非组件逻辑的合并，但是了解了组件patch过程后我们就发现并不是这样，Vue是永远伴随着组件而存在的，_init总是伴随着new Vue或者new Vue的子构造器而执行的，所以上面两种合并都是组件选项的合并。不同的是else逻辑是我们手动去调用new Vue时候执行的，比如我们创建根组件时候，而if逻辑则是我们再挂载过程中遇到子组件的patch才执行的，可以说是他们的调用方式是不同的。
  关于这部分内容我们在这里简单分析一下，为了避免章节内容过多，我们把它放到mergeOptions文档中详细分析，我们先写一个小的dome来更好的解读它。

let childComp = {
  template: '<div>{{msg}}</div>',
  created() {
    console.log('child create')
  },
  mounted() {
    console.log('child mount')
  },
  data() {
    return {
      msg: 'Hello Vue'
    }
  }
}

Vue.mixin({
  created() {
    console.log('parent create')
  }
})

new Vue({
  el: '#app',
  render: h => h(childComp)
})

  如上所示我们在执行new Vue时候执行的是else逻辑的合并，它通过mergeOptions方法来合并选项，该方法传入两个参数resolveConstructorOptions()和options。其中resolveConstructorOptions()又传入了vm.constructor，也就是我们的Vue，这个方法主要是处理一些在组件构造函数已经创建后应用全局混入问题，他会针对全局混入再做一次合并，如果你足够细心就会发现在createComponent方法中创建组件vnode时也调用了这个方法，当然这里我们没有涉及到子构造器，它就是简单返回了Vue.options。
  那Vue.options又是什么东西呢？在global-api/index.js中initGlobalAPI函数定义了它，这个之前我们应该有提到。既然提到了global-api，我们不妨再看看Vue.mixin做了些什么。initGlobalAPI函数最后执行了initMixin(Vue)。我们来看initMixin定义，其定义于global-api/mixin.js中

import { mergeOptions } from '../util/index'

export function initMixin (Vue: GlobalAPI) {
  Vue.mixin = function (mixin: Object) {
    this.options = mergeOptions(this.options, mixin)
    return this
  }
}

  其逻辑实际上也十分简单就是将Vue.options与mixin对象进行合并，然后返回Vue,这里this指向就是Vue，至于它的合并函数其实和_init方法中的合并函数是一样的。你可能觉得上面的讲解太浅显，不要担心，我们把详细的讲解都放在合并选项文档里了。
