# 2.1 new Vue()时做了什么

由上节内容可知new Vue() 时执行了this._init()方法，实际上我们看到Vue定义时并没有_init()方法，那么只能是instance/index文件下的Mixin方法混入了，很快我们便找到在initMixin()函数中找到了Vue.prototype上挂载的_init方法。

initMixin()定义于instance/init.js文件下

let uid = 0
export function initMixin (Vue: Class<Component>) {
  Vue.prototype._init = function (options?: Object) {
    const vm: Component = this // --> Vue构造的对象 
    vm._uid = uid++
    let startTag, endTag
    // a flag to avoid this being observed
    vm._isVue = true
    // merge options 合并option
    if (options && options._isComponent) {
      // optimize internal component instantiation 优化内部组件实例化
      // since dynamic options merging is pretty slow, and none of the 因为动态选项合并非常慢，而且
      // internal component options needs special treatment. 内部组件选项需要特殊处理
      initInternalComponent(vm, options)
    } else {
      vm.$options = mergeOptions(
        resolveConstructorOptions(vm.constructor),
        options || {},
        vm
      )
    }
    /* istanbul ignore else */
    if (process.env.NODE_ENV !== 'production') {
      initProxy(vm)
    } else {
      vm._renderProxy = vm
    }
    // expose real self
    vm._self = vm
    initLifecycle(vm)
    initEvents(vm)
    initRender(vm)
    callHook(vm, 'beforeCreate')
    initInjections(vm) // resolve injections before data/props
    initState(vm)
    initProvide(vm) // resolve provide after data/props
    callHook(vm, 'created')
    if (process.env.NODE_ENV !== 'production' && config.performance && mark) {
      vm._name = formatComponentName(vm, false)
      mark(endTag)
      measure(`vue ${vm._name} init`, startTag, endTag)
    }
    if (vm.$options.el) {
      vm.$mount(vm.$options.el)
    }
  }
}

开始声明uid = 0，在initMixin中Vue原型上定义了_init方法，该方法首先声明vm变量等于this 即Vue构造的对象，然后将uid ++ 赋值给vm.uid，之后对options合并做了优化处理，然后将合并后的options赋值给了vm.$options。在这里我们简单分析一下options的合并。mergeOptions简析

Vue合并options后在生产环境下将vm赋值给vm._renderProxy，而在开发环境则执行了initProxy(vm)方法。initProxy方法详解

之后初始化生命周期，初始化事件方法，初始化Render，调用beforeCreate钩子，初始化initInjections, 初始化State，初始化Provide，调用created钩子，最后调用vm.$mount()方法完成页面渲染。

Vue在初始化时执行了很多方法，new Vue()时候执行很多初始化方法，Vue代码设计脉络十分清晰，它就像一颗树一样，主干是十分清楚的，但是它也同树一样有错综复杂的枝干，函数层层嵌套，我们分析Vue要顺着一条主线去研究，而不是一行行代码去分析，看到一个函数就点进去深入分析它，这样深度遍历，是非常累而且没有什么必要的事，这样学习你会发现Vue越看越复杂，可能你连Vue的定义都没搞清楚的时候你已经去分析一堆函数的实现，这样导致你根本看不懂这些函数做了什么，而且不同于树结构，同一个函数可能有多个地方的引用，对不同情况下的实现不同，导致我们完全懵掉。所以我们分析Vue要跟着一条脉络去研究，有目的的去研究，不相关的细枝末节都暂时砍掉，这样会使我们思维更加清晰，更加容易理解。比如我们想要研究Vue是怎么将内容挂载到页面上的，我们就只需要分析挂载有关的逻辑，而不去关心其他关系较小或者无关的逻辑，这部分逻辑我们只需要浅析一下，了解其大致功能即可。
