  vue-router是官方的路由插件，它和vue.js 是深度集成的，我们大多数情况会用vue来开发一个单页应用，这就少不了去使用vue-router，它能帮助我们去管理URL，从而根据不同的URL去渲染不同的组件，如果没有vue-router，我们开发单页应用会复杂的多。接下来我们就来看看vue-router是怎么设计的吧！

# 8.1 vue-router的注册

  我们在3.6小节讲解组件的注册流程时，我们了解了Vue的全局api，包括Vue.use方法，这个方法是专门用来注册插件的。我们在注册路由插件时通常会在main.js中去使用use方法去注册router。
  Vue.use方法实际上是调用plugin的install方法去注册插件，所以我们必须要找到vue-router的install方法，在这之前我们先要找到vue-router的入口。
它的入口文件在src/index.js

import VueRouter from './entries/cjs'

export default VueRouter

  只有短短的两行代码，实际上VueRouter是指向./router.js的，在router.js中我们发现router是一个大的class。

  我们可以看到第一个定义的静态方法就是install方法，然后我们继续放下找果然发现install方法的定义

VueRouter.install = install
VueRouter.version = '__VERSION__'
VueRouter.isNavigationFailure = isNavigationFailure
VueRouter.NavigationFailureType = NavigationFailureType
VueRouter.START_LOCATION = START

if (inBrowser && window.Vue) {
  window.Vue.use(VueRouter)

  这里定义给几个静态属性赋值，如果在浏览器环境并且Vue是挂载到window上时，就直接调用window.Vue.use进行注册。当然正常来说Vue是不会绑定到window对象上的。接下来我们看看install方法定义。
  install方法定义在install.js文件下

import View from './components/view'
import Link from './components/link'

export let _Vue
export function install (Vue) {
  if (install.installed && _Vue === Vue) return
  install.installed = true
  _Vue = Vue
  const isDef = v => v !== undefined
  const registerInstance = (vm, callVal) => {
    let i = vm.$options._parentVnode
    if (isDef(i) && isDef(i = i.data) && isDef(i = i.registerRouteInstance)) {
      i(vm, callVal)
    }
  }
  Vue.mixin({
    beforeCreate () {
      registerInstance(this, this)
    },
    destroyed () {
      registerInstance(this)
    }
  })
  Object.defineProperty(Vue.prototype, '$router', {
    get () { return this._routerRoot._router }
  })
  Object.defineProperty(Vue.prototype, '$route', {
    get () { return this._routerRoot._route }
  })
  Vue.component('RouterView', View)
  Vue.component('RouterLink', Link)
  const strats = Vue.config.optionMergeStrategies
  // use the same hook merging strategy for route hooks
  strats.beforeRouteEnter = strats.beforeRouteLeave = strats.beforeRouteUpdate = strats.created
}

  可以看到install方法首先判断install.installed && _Vue === Vue，如果满足这个条件就直接return，因为满足这个条件的话说明路由已经被注册了，所以就不需要进行重复注册了，然后就是把install.installed = true。接下来定义了两个方法isDef和registerIntance方法，这两个方法我们先不看，接下来我们看Vue.mixin方法，这里通过mixin方法混入了beforeCreate钩子和destroyed钩子，这两个钩子就保证了我们组件中都能够访问到路由对象。
下面我们来看mixin做了什么事

beforeCreate () {
  if (isDef(this.$options.router)) {
    this._routerRoot = this
    this._router = this.$options.router
    this._router.init(this)
    Vue.util.defineReactive(this, '_route', this._router.history.current)
  } else {
    this._routerRoot = (this.$parent && this.$parent._routerRoot) || this}
  registerInstance(this, this)
},

  这里我们发现它去获取this.$options.router，这个东西是哪里来的呢，其实是我们在注册路由时会将路由对象传入到root Vue实例上，如下所示：

new Vue({
  el: '#app',
  render: h => h(App),
  router
})

  这里我们传入了router，这时我们就可以在new Vue时执行beforeCreate方法，这样就会执行if逻辑，因为这时可以获取到router对象。接下来要做的事也很简单，就是把this._routerRoot = this，也就保留我们根组件实例，然后用_router来保留我们传入的路由对象，然后就是调用router.init方法，router对象其实就是实例化VueRouter类得到的，接下来是调用Vue.util.defineReactive方法去把_router变为响应式对象。我们没有传入router对象时就会执行else逻辑，else逻辑主要是为非根组件实例工作的，这部分逻辑确保所有组件都能访问到this._routerRoot对象。它给所以组件添加this._routerRoot，并且如果有父组件的话就从父组件去取值，这样就保证_routerRoot可以一级一级向下传递。最后registerInstance方法是为router-view组件服务的，我们暂时不去分析它。
  接下来是通过defineProperty方法添加$router和$route对象，$router对象很好理解，就是我们实例化的路由对象，但是$route是什么时候添加的我们还不清楚，我们先继续往下看，接下来是通过Vue.component方法注册了router-view和router-link组件。最后是拿到route的合并策略并设置为相同
