# 7.4 transition（中）

  上小节我们分析了transition组件的定义，发现它并没有实现任何动画的能力，只是简单的为vnode.data添加了transition属性，实际上transition组件的实现依赖的是一个modules，在runtime下的modules中有个transition.js

export default inBrowser ? {
  create: _enter,
  activate: _enter,
  remove (vnode: VNode, rm: Function) {
    /* istanbul ignore else */
    if (vnode.data.show !== true) {
      leave(vnode, rm)
    } else {
      rm()
    }
  }
} : {}

  这个文件会导出这些hook方法，我们知道在patch时会执行invokeCreateHooks方法

  function invokeCreateHooks(vnode, insertedVnodeQueue) {
    for (let i = 0; i < cbs.create.length; ++i) {
      cbs.create[i](emptyNode, vnode)
    }
    i = vnode.data.hook // Reuse variable
    if (isDef(i)) {
      if (isDef(i.create)) i.create(emptyNode, vnode)
      if (isDef(i.insert)) insertedVnodeQueue.push(vnode)

  }

  这里的cbs.create其实就是这里create钩子，也就是_enter方法的调用。接下来看这个方法到底做了什么事

function _enter (_: any, vnode: VNodeWithData) {
  if (vnode.data.show !== true) {
    enter(vnode)
  }
}

  这里会判断data.show不等于true，我们知道transition也可以用做v-show指令的切换动画。但是v-show会做单独的处理，所以这里要排除它。

export function enter (vnode: VNodeWithData, toggleDisplay: ?() => void) {}

  这里执行enter方法，它接收一个vnode和一个toggleDisplay函数，但这里只传入了vnode。

  首先它获取到vnode对应dom元素，这个时候我们真实dom已经被createElm方法创建了，所以是可以获取到真实dom的。
  然后下面这个el._leaveCb函数的逻辑我们先不看。这个东西在下面我们就会看到它是怎么定义的。

  const data = resolveTransition(vnode.data.transition)

  接下来会执行resolveTransition方法生成data，这里就会把我们transition组件生成的vnode.data.transition给传入，所以这部分应该是个重点逻辑。

export function resolveTransition (def?: string | Object): ?Object {
  if (!def) {
    return
  }
  /* istanbul ignore else */
  if (typeof def === 'object') {
    const res = {}
    if (def.css !== false) {
      extend(res, autoCssTransition(def.name || 'v'))
    }
    extend(res, def)
    return res
  } else if (typeof def === 'string') {
    return autoCssTransition(def)
  }
}

  首先没有def就直接return。   然后def是个object类型话就初始化一个res对象，如果def.css没有被显式定义为false的话就执行。css为我们一个props选项，如果设置为false那么动画就不能通过css触发，而是要通过js钩子。   所以这个内部逻辑应该是对css的处理，我们来看autoCssTransition方法是怎样定义的

const autoCssTransition: (name: string) => Object = cached(name => {
  return {
    enterClass: `${name}-enter`,
    enterToClass: `${name}-enter-to`,
    enterActiveClass: `${name}-enter-active`,
    leaveClass: `${name}-leave`,
    leaveToClass: `${name}-leave-to`,
    leaveActiveClass: `${name}-leave-active`
  }
})

  其实是定义比较简单，如果我们transition定义了name，那么会添加这些类名，类名是name拼接一些字符串，其实这些我们都应该使用过程，比如我name定义为'fade'，那么我们定义类名时会定义为fade-enter、fade-enter-active等，这个功能就是这里实现的。如果我们不定义name那么默认name为v。
  我们回到resolveTransition函数中，它会把autoCssTransition返回的对象都合并到res中。然后会将def上的内容也合并到res，最后返回res。
  接下来是判断def是一个string，这个操作有点迷，但是可能不是为这里的逻辑准备的。如果def是string的话它是当做一个name去处理的。

  如果el._enterCb存在或者不是一个元素节点就直接return。然后从data中解构一堆变量以供后面使用。

  // transition. One edge case to check is when the <transition> is placed
  // as the root node of a child component. In that case we need to check
  // <transition>'s parent for appear check.
  let context = activeInstance
  let transitionNode = activeInstance.$vnode
  while (transitionNode && transitionNode.parent) {
    context = transitionNode.context
    transitionNode = transitionNode.parent

  这里先获取activeInstance，这个是定义在lifecycle中，我们很早以前就分析过了，它表示正在激活的组件，实际上就是正在patch的组件。其实就是transition组件，然后获取它的$vnode保存在transitionNode变量中，也就组件vnode。
  接下来是一个while循环，如果transitionNode.parent存在说明它是组件的一个根节点，这时候会把content设置为transitionNode.context。transitionNode赋值为transitionNode.parent。
  这段逻辑是为了处理边界情况的，如果transition为一个组件的根节点那么会执行这段代码。再往下如果context还没有mounted的话或者不是组件根vnode的话会将isAppear赋值为true，接下来如果isAppear为true，并且!appear并且appear不等于空字符串就直接return。isAppear是标识是否能添加出现动画。
  其实我们后面逻辑就是对过渡和动画的处理，如果isAppear为true说明没有加载完成，并且我们也没有设置appear属性，那么我们就直接返回不执行后面逻辑。我们知道appear是一个开启首次出场动画的标识。再来看接下来会执行什么逻辑。

  const startClass = isAppear && appearClass
    ? appearClass
    : enterClass
  const activeClass = isAppear && appearActiveClass
    ? appearActiveClass
    : enterActiveClass
  const toClass = isAppear && appearToClass
    ? appearToClass
    : enterToClass

  const beforeEnterHook = isAppear
    ? (beforeAppear || beforeEnter)
    : beforeEnter
  const enterHook = isAppear
    ? (typeof appear === 'function' ? appear : enter)
    : enter
  const afterEnterHook = isAppear
    ? (afterAppear || afterEnter)
    : afterEnter
  const enterCancelledHook = isAppear
    ? (appearCancelled || enterCancelled)
    : enterCancelled

其实就是处理了一些出场动画的class和js钩子

  const explicitEnterDuration: any = toNumber(

      ? duration.enter
      : duration

  if (process.env.NODE_ENV !== 'production' && explicitEnterDuration != null) {

  处理进场动画时间，看官方文档就可以知道，是有两种写法，一种对象配置，可以分别控制进场和出场时间，另一种是直接配置时间。然后会对配置的合法性做一个检查

function checkDuration (val, name, vnode) {
  if (typeof val !== 'number') {
    warn(
      `<transition> explicit ${name} duration is not a valid number - ` +
      `got ${JSON.stringify(val)}.`,
      vnode.context
    )
  } else if (isNaN(val)) {
    warn(
      `<transition> explicit ${name} duration is NaN - ` +
      'the duration expression might be incorrect.',
      vnode.context
    )
  }
}

接着往下

  const expectsCSS = css !== false && !isIE9
  const userWantsControl = getHookArgumentsLength(enterHook)

  如果css没有被显式定义为false并且不是在ie9环境下expectsCSS为true。接下来是定义userWantsControl，这个变量是来判断用户是否想要控制enterHook钩子。我们来看getHookArgumentsLength

function getHookArgumentsLength (fn: Function): boolean {
  if (isUndef(fn)) {
    return false
  }
  const invokerFns = fn.fns
  if (isDef(invokerFns)) {
    // invoker
    return getHookArgumentsLength(
      Array.isArray(invokerFns)
        ? invokerFns[0]
        : invokerFns
    )
  } else {
    return (fn._length || fn.length) > 1
  }
}

  如果没有的enterHook函数的话就什么都不做。
  如果有fn的话就会取fn.fns，因为我们事件绑定都会被invoker函数包裹，其真正执行函数在其fns属性上。
  如果invokerFns存在就递归调用getHookArgumentsLength，如果它是个数组就取数组第一个元素作为参数传入，直到fn.fns不存在为止，那么它就是一个单纯的执行函数，然后去返回它的length并判断其是不是大于1。

  enter: function (el, done) {
    // ...
    done()
  }

  这个是我们对enter函数定义，如果和css动画配合使用时，会通过css定义的时间来决定enter的结束，但是如果接收done函数的话也会允许我们手动去控制enter结束，那么这里done是可选的，如果我们这里定义时候不去接收done参数，那么fn.length就是1，那么可以断定用户不会去尝试控制它。反之则认为用户可能会想要控制enter的结束，也就是length > 1的情况。

  const cb = el._enterCb = once(() => {
    if (expectsCSS) {
      removeTransitionClass(el, toClass)
      removeTransitionClass(el, activeClass)
    }
    if (cb.cancelled) {

        removeTransitionClass(el, startClass)

    } else {
      afterEnterHook && afterEnterHook(el)
    }
    el._enterCb = null
  })

  这里就是给_enterCb方法赋值了，我们前面知道如果它存在的话就会直接return，所以这个方法应该是关键逻辑。打但是它被once定义，也就是说它只会执行一次，执行后它会将_enterCb重置。
  我们来看removeTransitionClass方法

export function removeTransitionClass (el: any, cls: string) {
  if (el._transitionClasses) {
    remove(el._transitionClasses, cls)
  }
  removeClass(el, cls)
}

  如果存在el._transitionClasses列表，就把cls从列表删除，然后会调用removeClass方法

export function removeClass (el: HTMLElement, cls: ?string) {
  if (!cls || !(cls = cls.trim())) {
    return
  }
  if (el.classList) {
    if (cls.indexOf(' ') > -1) {
      cls.split(whitespaceRE).forEach(c => el.classList.remove(c))
    } else {
      el.classList.remove(cls)
    }
    if (!el.classList.length) {
      el.removeAttribute('class')
    }
  } else {
    let cur = ` ${el.getAttribute('class') || ''} `
    const tar = ' ' + cls + ' '
    while (cur.indexOf(tar) >= 0) {
      cur = cur.replace(tar, ' ')
    }
    cur = cur.trim()
    if (cur) {
      el.setAttribute('class', cur)
    } else {
      el.removeAttribute('class')
    }
  }
}

  首先如果cls为空就什么都不做。
  如果el.classList存在：先判断cls中间是否有空格，如果有的话，说明是多个类名的字符串，那么通过split分割为数组，并把这些类名都移除。如果是单个类名那么直接移除就可以了。
  如果el.classList不存在：那就是浏览器不支持classList，那就降级采用getAttribute方法去获取类名，如果没有的话它会返回null，所以我们要将它初始化为空字符串，注意我们这里用模板字符串给cur前后都加了一个空格。然后我们给cls也前后加个空格保存到tar中，然后如果cur中找到tar时就将其替换为一个空格并即将cur重新赋值，直到找不到tar为止。通过这种巧妙的方法将类名给删除。最后将cur前后空格去掉。如果cur存在那就给重新设置class属性，否则删除class。
  回到_enterCb函数中，这里会先删除toClass，对应v-enter，然后会删除activeClass，对应v-enter-to。
  如果cb.canceled存在也就是过渡被取消，那就将过渡开始钩子删除，然后如果取消的钩子被定义的话还要调用enterCancelledHook取消的钩子函数。如果过渡没有取消你那么就执行afterEnterHook。

  if (!vnode.data.show) {
    // remove pending leave element on enter by injecting an insert hook
    mergeVNodeHook(vnode, 'insert', () => {
      const parent = el.parentNode
      const pendingNode = parent && parent._pending && parent._pending[vnode.key]
      if (pendingNode &&
        pendingNode.tag === vnode.tag &&
        pendingNode.elm._leaveCb
      ) {
        pendingNode.elm._leaveCb()
      }
      enterHook && enterHook(el, cb)
    })
  }

  如果没有使用v-show，那么会往insert钩子中添加一个钩子函数。这部分逻辑主要是执行enterHook函数，这里会传入el和cb，所以cb实际上就是我们done函数。

  先执行beforeEnterHook，然后添加startClass、activeClass，nextFrame后将startClass移除，如果没有被取消的话会添加toClass类 。如果没有用户控制过渡结束的话那么就根据定义的过渡时间去设置一个定时器，定时器结束后调用我们cb函数。cb函数就是上面我们定义的el._enterCb函数，会将toClass、activeClass删除并且取调用afterEnterHook。
  这里会先校验用户传入的过渡时间，通过isValidDuration方法

function isValidDuration (val) {
  return typeof val === 'number' && !isNaN(val)
}

  如果用户定义时间不合法或者没有传入时间，那么就执行else逻辑，也就是会执行whenTransitionEnds，这里我们有好几个新函数没有去分析，接下来我们来分析一下这些函数，首先看addTransitionClass方法

export function addTransitionClass (el: any, cls: string) {
  const transitionClasses = el._transitionClasses || (el._transitionClasses = [])
  if (transitionClasses.indexOf(cls) < 0) {
    transitionClasses.push(cls)
    addClass(el, cls)
  }
}

  相对于我们之前分析的移除方法，那这里就是添加class方法，主要通过addClass方法来添加类名

export function addClass (el: HTMLElement, cls: ?string) {
  if (!cls || !(cls = cls.trim())) {
    return
  }
  if (el.classList) {
    if (cls.indexOf(' ') > -1) {
      cls.split(whitespaceRE).forEach(c => el.classList.add(c))
    } else {
      el.classList.add(cls)
    }
  } else {
    const cur = ` ${el.getAttribute('class') || ''} `
    if (cur.indexOf(' ' + cls + ' ') < 0) {
      el.setAttribute('class', (cur + cls).trim())
    }
  }
}

这和我们之前的移除方法是别无二致的。这里就不赘述了。 然后来看nextFrame方法

export function nextFrame (fn: Function) {
  raf(() => {
    raf(fn)
  })
}

  这里是调用raf方法，然后传入一个回调又调用了raf方法并将fn作为参数传入。

// binding to window is necessary to make hot reload work in IE in strict mode
// 绑定到窗口对于在IE中以严格模式进行热重新加载是必要的
const raf = inBrowser
  ? window.requestAnimationFrame
    ? window.requestAnimationFrame.bind(window)
    : setTimeout
  : /* istanbul ignore next */ fn => fn()

raf的本质是window.requestAnimationFrame，可以实现在下一帧时调用fn。也就是执行回调函数中移除startClass、添加toClass的逻辑。而过渡完成后会调用cb函数删除toClass、activeClass。对应我们官网描述：

  1. v-enter：定义进入过渡的开始状态。在元素被插入之前生效，在元素被插入之后的下一帧移除。
  2. v-enter-active：定义进入过渡生效时的状态。在整个进入过渡的阶段中应用，在元素被插入之前生效，在过渡/动画完成之后移除。这个类可以被用来定义进入过渡的过程时间，延迟和曲线函数。
  3. v-enter-to：定义进入过渡的结束状态。在元素被插入之后下一帧生效 (与此同时 v-enter 被移除)，在过渡/动画完成之后移除。 然后我们还有whenTransitionEnds方法没有分析，它是用户没有定义过渡时间的情况，但是可以肯定的是它也是会调用我们的cb函数完成我们上述流程。
  if (vnode.data.show) {
    toggleDisplay && toggleDisplay()
    enterHook && enterHook(el, cb)
  }

  if (!expectsCSS && !userWantsControl) {
    cb()
  }

  最后如果!expectsCSS && !userWantsControl会调用cb()，因为这样能保证我们afterEnterHook函数能被执行，而userWantsControl为true时，我们done函数就是我们的cb函数，所以可以实现手动调用。
