# 7.3 transition（上）

  这个小节我们来分析另一个常用的内置组件transition，这是Vue用开为我们管理过渡和动画效果的组件。
  它的定义在platforms/web/runtime/components文件夹下，可以看到

export default {
  name: 'transition',
  props: transitionProps,
  abstract: true,

  render (h: Function) {
  }
}

这个组件由这几部分构成，我们先来看props定义，props是定义了比较多的内容

export const transitionProps = {
  name: String,
  appear: Boolean,
  css: Boolean,
  mode: String,
  type: String,
  enterClass: String,
  leaveClass: String,
  enterToClass: String,
  leaveToClass: String,
  enterActiveClass: String,
  leaveActiveClass: String,
  appearClass: String,
  appearActiveClass: String,
  appearToClass: String,
  duration: [Number, String, Object]
}

  关于这个几个props我们只分析其中一些，对于类名相关我们就去分析了。
  下面是来自官网对这几个props的描述
  name - string，用于自动生成 CSS 过渡类名。例如：name: 'fade' 将自动拓展为 .fade-enter，.fade-enter-active 等。默认类名为 "v"。
  appear - boolean，是否在初始渲染时使用过渡。默认为 false。设置为true后第一次进入页面也会触发过渡动画。
  css - boolean，是否使用 CSS 过渡类。默认为 true。如果设置为 false，将只通过组件事件触发注册的 JavaScript 钩子。
  type - string，指定过渡事件类型，侦听过渡何时结束。有效值为 "transition" 和 "animation"。默认 Vue.js 将自动检测出持续时间长的为过渡事件类型。
  mode - string，控制离开/进入过渡的时间序列。有效的模式有 "out-in" 和 "in-out"；默认同时进行。如果我们过渡涉及新dom插入和旧dom的删除，那么可以通过设置mode来调整顺序，比如设置为out-in，就会先等待移除动画完成后再执行进入动画。
  duration - number | { enter: number, leave: number } 指定过渡的持续时间。默认情况下，Vue 会等待过渡所在根元素的第一个 transitionend 或 animationend 事件。
  这个组件也是个抽象组件，也就是不会渲染到真实dom，接下来我们看它的render函数实现。

  let children: any = this.$slots.default
  if (!children) {
    return
  }

  children = children.filter(isNotTextNode)
  /* istanbul ignore if */

  首先获取默认插槽，如果没有获取到则直接返回，其实我们这些内置组件实际上都是依赖插槽去实现的。如果没有在组件中定义内容那么也就什么都不需要做了。如果获取到children要对它做一个过滤，isNotTextNode译为非文本节点，也就是说我们要去找元素节点，因为单单的文本不存在过渡。
const isNotTextNode = (c: VNode) => c.tag || isAsyncPlaceholder(c)   我们再来看isAsyncPlaceholder，这个方法我们见过很多次了，从名字就可以看出它是做什么用的，所以我们一直没有贴出它的源码

export function isAsyncPlaceholder (node: VNode): boolean {
  return node.isComment && node.asyncFactory
}

  其实就是判断是否是异步组件的占位节点。
  这里将children过滤后如果发现为空，那就直接return，因为没有任何意义。如果不为空的话再判断其length是否大于1，如果大于1的话会报一个警告，因为我们的transition组件仅允许使用单个节点，如果是多个节点，应该使用transition-group组件，这也是Vue为我们提供的能力。

hasParentTransition如下所示

function hasParentTransition (vnode: VNode): ?boolean {
  while ((vnode = vnode.parent)) {
    if (vnode.data.transition) {
      return true
    }
  }
}

  可以看到实际上就是判断vnode.parent是否也是transition的vnode，也就是transition中直接嵌套另一个transition。vnode.parent只有渲染vnode才存在，渲染vnode就是组件下根元素的vnode。这里vnode.data.transition是什么我们后面就会说明。如果是这种情况直接返回rawChild。

  // apply transition data to child
  // use getRealChild() to ignore abstract components e.g. keep-alive
  const child: ?VNode = getRealChild(rawChild)
  /* istanbul ignore if */
  if (!child) {
    return rawChild
  }
  if (this._leaving) {
    return placeholder(h, rawChild)
  }

  这里会获取真正的子元素，因为rawChild可能还是个抽象组件，那么它就不是真实可渲染的节点。getRealChild如下

// in case the child is also an abstract component, e.g. <keep-alive>
// we want to recursively retrieve the real component to be rendered
// 如果子组件也是抽象组件，例如<keep-alive>，我们希望递归地检索要呈现的真实组件
function getRealChild (vnode: ?VNode): ?VNode {
  const compOptions: ?VNodeComponentOptions = vnode && vnode.componentOptions
  if (compOptions && compOptions.Ctor.options.abstract) {
    return getRealChild(getFirstComponentChild(compOptions.children))
  } else {
    return vnode
  }
}

  如果发现抽象组件就递归去找子组件，并且取抽象组件下的第一个子组件，否则就直接返回vnode。如果最后找不到子组件就那么满足!child，那么就返回rawChild。
  接下来this._liveing存在执行placeholder方法

function placeholder (h: Function, rawChild: VNode): ?VNode {
  if (/\d-keep-alive$/.test(rawChild.tag)) {
    return h('keep-alive', {
      props: rawChild.componentOptions.propsData
    })
  }
}

  这也是对keep-alive逻辑的处理，如果tag是以数字后面拼-keep-alive结尾的，那么会返回一个h函数创建keep-alive，并将props传入。

// ensure a key that is unique to the vnode type and to this transition
// component instance. This key will be used to remove pending leaving nodes
// during entering.
// 确保vnode类型和此转换组件实例的密钥是唯一的。
// 此键将用于在进入期间删除挂起的离开节点。
const id: string = `__transition-${this._uid}-`
child.key = child.key == null
  ? child.isComment
    ? id + 'comment'
    : id + child.tag
  : isPrimitive(child.key)
    ? (String(child.key).indexOf(id) === 0 ? child.key : id + child.key)
    : child.key

const data: Object = (child.data || (child.data = {})).transition = extractTransitionData(this)

  接下来定义一个id，然后为child添加key，如果child是注释节点就拼接id和comment，否则拼接id和tag名称。
  如果key已经存在了，那就先判断key是不是isPrimitive，也就是原始类型数据。如果不是原始类型直接返回child.key，否则还要看key前缀是我们id是否一致，如果一致就直接用child.key，否则会在前面拼接一个id。这样可以保证key的唯一性。
  接下来就是关键逻辑给data.transition赋值

export function extractTransitionData (comp: Component): Object {
  const data = {}
  const options: ComponentOptions = comp.$options
  // props
  for (const key in options.propsData) {
    data[key] = comp[key]
  }
  // events.
  // extract listeners and pass them directly to the transition methods
  const listeners: ?Object = options._parentListeners
  for (const key in listeners) {
    data[camelize(key)] = listeners[key]
  }
  return data
}

  这里创建data对象，然后将propsData的内容添加到data中，然后将事件监听获取到添加到data中。我们知道transition组件是有很多JavaScript钩子函数可以监听的，这里就是获取这些事件。最后返回data就是我们的transition。

  const oldRawChild: VNode = this._vnode
  const oldChild: VNode = getRealChild(oldRawChild)
  // mark v-show
  // so that the transition module can hand over the control to the directive
  // 标记v-show，以便转换模块可以将控制权移交给指令
  if (child.data.directives && child.data.directives.some(isVShowDirective)) {

  接下来会获取oldRawChild和oldChild ，然后这里会对v-show做一个特殊标记，isVShowDirective方法如下：

const isVShowDirective = d => d.name === 'show'

然后接下来是一个复杂的判断

  if (
    oldChild &&
    oldChild.data &&
    !isSameChild(child, oldChild) &&
    !isAsyncPlaceholder(oldChild) &&
    // #6687 component root is a comment node
    !(oldChild.componentInstance && oldChild.componentInstance._vnode.isComment)
  ) {
    // replace old child transition data with fresh one
    // important for dynamic transitions!
    const oldData: Object = oldChild.data.transition = extend({}, data)
    // handle transition mode
    if (mode === 'out-in') {
      // return placeholder node and queue update when leave finishes
      this._leaving = true
      mergeVNodeHook(oldData, 'afterLeave', () => {
        this._leaving = false
        this.$forceUpdate()
      })
      return placeholder(h, rawChild)
    } else if (mode === 'in-out') {
      if (isAsyncPlaceholder(child)) {
        return oldRawChild
      }
      let delayedLeave
      const performLeave = () => { delayedLeave() }
      mergeVNodeHook(data, 'afterEnter', performLeave)
      mergeVNodeHook(data, 'enterCancelled', performLeave)
      mergeVNodeHook(oldData, 'delayLeave', leave => { delayedLeave = leave })
    }
  }

  如果oldChild.data存在。并且新旧child不同，并且旧的child不是异步占位节点，并且旧节点不是注释节点那么会执行内部逻辑。
  首先会将旧节点的transition更新。
  然后会处理mode，如果mode为'out-in'，会将this._leaving置为true，然后合并一个afterLeave钩子。这个钩子函数会将this._leaving重置并且强制更新。如果子组件是keep-alive的话会返回一个占位节点。
  如果mode为'in-out'，则如果child是异步占位节点就直接返回oldRawChild。接下来会定义一个delayedLeave，但是不赋值。再定义一个执行delayedLeave函数的函数performLeave，接下来会把performLeave函数添加到data的afterEnter和enterCancelled钩子上。但是这时候delayedLeave是没有值的，它的赋值是在最后，最后会为'delayLeave'添加一个修改delayedLeave的函数钩子，注意这个钩子是添加到oldData中的，oldData不是一个vnode，那么在执行mergeVNodeHook时就会将钩子函数直接添加到oldData上，而不是vnode.data.hook中。这里设计比较复杂，我们后面再来说明为什么这么设计。
render函数最后一步会返回rawChild
